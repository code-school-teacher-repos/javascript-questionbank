You are creating a form that asks users about their favorite pets. One of the elements on the form is the user's follow up email address.

You need to change the text of the error message that shows when the email is invalid.

Which function should you use?

setCustomValifity
validity.valueMissing
validity.typeMismatch
validity.customError

Explanation

You should use the setCustomValifity() function to change the text of the error message on the email. When you set the error message, the element is considered invalid. The error will be cleared when you set the value to an empty string. Here is an example:

<script>
      var mail = document.getElementById("mail");
      phone.addEventListener("input", function (event) {
           if (mail.validity.typeMismatch) {
                mail.setCustomValidity("I am expecting a valid email");
           } else {
                mail.setCustomValidity("");
           }
      });
</script>

The other options are incorrect because they are properties, not functions. They operate as follows:

validity.valueMissing: This property returns true if the element is required and has no value. It returns false otherwise.

validity.typeMismatch: It returns true if the value does not have the correct syntax. It returns false otherwise.

validity.customError: It returns true if the element has a custom error, but it does not set the custom error.

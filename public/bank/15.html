An HTML page references three JavaScript files as follows:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <div>This lesson illustrates working with scripts</div>
      <div id="result"></div>
      <script>
           var data = 'user@gmail.com';
           if (validate(data))
                document.getElementById("result").innerHTML = doAction(data);
      </script>
      <script src="Scripts/Common3.js"></script>
      <script src="Scripts/Common2.js"></script>
      <script src="Scripts/Common1.js"></script>
</body>
</html>

Common1.js



function checkInput(field) {
      if (!field)
           return false;

      const MAX = 20;
      return field.length <= 20;
}

Common2.js



function doAction(data) {
      if (!data)
           return '';
      var newData = '';
      for (var i = data.length - 1; i >= 0; i--) {
           newData += data[i];
      }
      return newData;
}

Common3.js



function validate(data) {
      return /^\w+([\.-]?\w+)*@\w+([\.-]?\w+)*(\.\w{2,3})+$/.test(data);
}

What will happen when the HTML page is loaded?

The browser window displays the following: This lesson illustrates working with scripts
The browser window displays the following: This lesson illustrates working with scripts moc.liamg@resu
The console displays a ReferenceError, and the browser window displays the following: This lesson illustrates working with scripts
An error is generated and nothing displays in the browser window.

Explanation

The console displays a ReferenceError, and the browser window displays the following:

This lesson illustrates working with scripts

In this scenario, the three JavaScript files are referenced at the bottom of the page. The first script declares a variable that contains an email address. The function validate() is checking whether the email address is valid or not using a regular expression. If the email address is valid, the function doAction() reverses the email address, which is then rendered in the result's inner HTML.
To start, the content of the first div "This lesson illustrates working with scripts" is rendered on the page anyway.

However, the file that contains the validate() function is referenced after the function is called. Therefore, when the execution reaches validate, it will not know what that function is. This causes a reference error to be displayed in the console window (F12).

Although the email address was valid, the validate() and doAction() functions are not recognized by the time the call was made to them (since the reference happened afterwards).

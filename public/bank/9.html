While reviewing another developer's code, you come across the following:

<div id="result" class="format-output"></div>
<script>
      /*
      This is an example of using functions. A function is a collection
      of statements with a name. You call the function by using its name.
      If the function needs additional information to do its work, you
      pass that information in the parenthesis after the function name.
      var name = 'John Doe';
      */
      var info = getName(name);
      document.getElementById("result").innerHTML = info;

      function getName(fullname) {
           if (!fullname)
                return 'empty';
           var index = fullname.indexOf(' ');
           if (index === -1)
                return 'no index';
           var sub = fullname.substring(index + 1);
           return sub;
      }
</script>

What will be displayed when you review this code in the browser?

John
no index
Doe
empty

Explanation

The browser will display the text empty. Commenting code is very important because it explains complex scenarios in the code and reflects the developer's intention. There are one line comments using // and multi-line comments using /**/. In this scenario, the multi-line comment is explaining the purpose of functions. However, unintentionally, the comment hid away the variable "name" declaration. So as far as the rest of the code is concerned, name is undefined.

The purpose of the function is to find the index of the space and return the second portion of the name. However, since the name is undefined, the first if statement's condition is true and "empty" is returned from the function.

If there were no space in the text "John Doe", then "no index" would have been returned by the function.

Neither "John" nor "Doe" can be returned by the function since the parameter name has not been initialized.

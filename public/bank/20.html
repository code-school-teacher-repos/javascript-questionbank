You have the following HTML page:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           try {
                processFile()
           }
           catch (ex) {
                console.log(ex);
           }
           finally {
                console.log('Closing the file')
           }

           function processFile(filename) {
                if (!filename)
                     throw new Error('No file to process!');
                console.log('Working on it');
                try {
                     console.log('Opening the file');
                     throw new Error('Issue with the file content');
                }
                catch (ex) {
                     console.log('File content not formatted properly');
                }
           finally {
                console.log('Wrapping up')
           }
      }
      </script>
</body>
</html>

When you display this page in the browser, what will be displayed in the console?

Error: No file to process! Closing the file
No file to process! Working on it Opening the file Issue with the file content File content not formatted properly Wrapping up
No file to process! Working on it Opening the file Issue with the file content File content not formatted properly Wrapping up Closing the file
Working on it Opening the file File content not formatted properly Wrapping up Closing the file

Explanation

The following will be displayed in the console:

Error: No file to process!
Closing the file

The script starts with a try/catch/finally block that calls a function to process a file (processFile()). Although this function takes a file name as a parameter, nothing is passed during the call.

The first line in the function checks whether there is a file name. If not, an error is thrown. That means the execution of the function will halt and control is passed to the caller (the try block above the function). Because an error is thrown, the catch block will execute. It logs the error to the console ("Error: No file to process!"), and the finally block will execute printing "Closing the file" in the console. Therefore, when loading this page in the browser, the following will show in the console (some browsers might print additional information such as stack trace):

Error: No file to process!
Closing the file

It is important to note that since the second line in the function threw an error, the rest of the code in the function body will not execute. The try/catch/finally blocks inside the function will not execute and none of the console log statements will be rendered in the console.

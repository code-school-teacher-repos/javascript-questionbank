SubObjective:

Complete and debug code that interacts with the Browser Object Model (BOM)

The following script displays information about the screen:

<script>
      var result =
           `screen.width = ${screen.width}
           screen.height = ${screen.height}
           screen.availWidth = ${screen.availWidth}
           screen.availHeight = ${screen.availHeight}
           screen.colorDepth = ${screen.colorDepth}
           screen.pixelDepth = ${screen.pixelDepth}`;
      alert(result);
</script>

For each of the following statements about these properties, select Yes if the statement is true. Otherwise, select No.

Explanation

The only option that is correct is:

screen.availHeight < screen.height

The window.screen object contains information about the screen. It has the following properties:

screen.width
screen.height
screen.availWidth
screen.availHeight
screen.colorDepth
screen.pixelDepth

The available width and available height are smaller than their corresponding width and height. The reason is that the actual width and height already account for the interface features displayed by the operating system. The taskbar is a good example. The available height will be the actual height minus the height occupied by the taskbar. This explains why the screen's available height (availHeight) is less than its height.

colorDepth returns the color depth of the screen, and the pixel depth gets the bit depth of the screen. For compatibility reasons, these options return the value 24 by specification. Because these values are the same, one is neither greater than nor less than the other.

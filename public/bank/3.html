SubObjective:

Complete or debug code that uses assignment and arithmetic operators

Evaluate the following statements:

var outcome1 = 23 > 19 && 4 + 7 > 12;
var outcome2 = 27;
var outcome3 = outcome2 /= 2;
console.log("%d\t%d", outcome1, outcome3);

What will be printed in the console?

0 13
false 13
true 1
false 13.5

Explanation

0 13 would be printed in the console. Order of operator evaluation is the key to determining the output. Dividing the expression 23 > 19 && 4 + 7 > 12 into two pieces will clarify the evaluation, and will result in the following:

23 > 19
4 + 7 > 12

23 is greater than 19. But does 4 + 7 > 12 mean (4 + 7) > 12 or 4 + (7 > 12)?

Looking at the table of operator precedence, + has higher precedence than >. Therefore, the expression is actually (4 + 7) > 12, which evaluates to false.

Putting the results together true && false evaluates to false because both of the operands must be true in order for the result to be true.

On the other hand, 27/2 = 13.5, which is stored in outcome3.

Finally, the console statement prints the output. However, it uses the %d, and that results in an integer value. The integer representation of false is 0 and that of 13.5 is 13. So, the final outcome is 0 13.

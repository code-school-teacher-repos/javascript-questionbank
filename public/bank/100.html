Interact with HTML Forms
SubObjective:

Complete and debug code that retrieves input from forms and sets form field values

You are creating a form to fill out employee information with the following requirements:

* When submitted, the form will submit to the same page.
* When the user clicks a button, the form data will be cleared out.

Which form tag and input elements should you use?

<form method="post" enctype="application/x-www-form-urlencoded"> <div>      <input type="submit" value="Submit">      <input type="reset" value="Reset"> </div>
<form action="http://localhost/default.html" method="post" enctype="application/x-www-form-urlencoded"> <div>      <input type="submit" value="Submit">      <input type="button" value="Reset"> </div>
<form method="post" enctype="application/x-www-form-urlencoded"> <div>      <input type="submit" value="Submit">      <input type="submit" value="Reset"> </div>
<form action="default" method="post" enctype="application/x-www-form-urlencoded"> <div>      <input type="submit" value="Submit">      <input type="reset" value="Reset"> </div>

Explanation

The form tag and input fields suitable for this scenario are:

<form method="post" enctype="application/x-www-form-urlencoded">

<div>
      <input type="submit" value="Submit">
      <input type="reset" value="Reset">
</div>

In the form tag, the action attribute is used to indicate where the form data is submitted. Since by default the data is submitted to the same page, there is no need to indicate the action attribute.

Also, you can indicate the type of input to be submit or reset. The submit button will submit the form data to the page specified in the action attribute. The default is the same page in this case.

The reset type can be used to clear out the field values. If you are filling out a long form, and for some reason you decided to start over, you can click on this button and all input elements will be cleared out. Note that the type needs to be reset and the value attribute is the text showing on the button.

You should not use the following form actions:

<form action="default" ...>

<form action=http://localhost/default.html ...>

The requirement is to submit to the same page by default, and the name of the page to submit to was not specified in the requirement.

Interact with HTML Forms
SubObjective:

Describe the form submission process

A form on an HTML page contains an input of type 'file'. The user selects a file and submits the form.

What will be the HTTP request header?

POST http://localhost:35132/Account/FormTest HTTP/1.1 Accept: text/html, application/xhtml+xml, image/jxr, */* Content-Type: multipart/form-data; Accept-Encoding: gzip, deflate Content-Length: 230 Host: localhost:35132 Content-Type: text/plain This is a test file.
POST http://localhost:35132/Account/FormTest HTTP/1.1 Accept: text/html, application/xhtml+xml, image/jxr, */* Content-Type: x-www-form-urlencoded; boundary=---------------------------7e12c72e16083e Accept-Encoding: gzip, deflate Content-Length: 230 Host: localhost:35132 -----------------------------7e12c72e16083e Content-Type: text/plain This is a test file. -----------------------------7e12c72e16083e--
The request header is empty.
POST http://localhost:35132/Account/FormTest HTTP/1.1 Accept: text/html, application/xhtml+xml, image/jxr, */* Content-Type: multipart/form-data; boundary=---------------------------7e12c72e16083e Accept-Encoding: gzip, deflate Content-Length: 230 Host: localhost:35132 -----------------------------7e12c72e16083e Content-Disposition: form-data; name="myFile"; filename="C:\Testing\test.txt" Content-Type: text/plain This is a test file. -----------------------------7e12c72e16083e--

Explanation

The request header will be:

POST http://localhost:35132/Account/FormTest HTTP/1.1
Accept: text/html, application/xhtml+xml, image/jxr, */*
Content-Type: multipart/form-data; boundary=---------------------------7e12c72e16083e
Accept-Encoding: gzip, deflate
Content-Length: 230
Host: localhost:35132

-----------------------------7e12c72e16083e
Content-Disposition: form-data; name="myFile"; filename="C:\Testing\test.txt"
Content-Type: text/plain

This is a test file.
-----------------------------7e12c72e16083e--

When you submit a file from a form, you need to make sure the following requirements are in place:

* You are able to select the file using a file picker. In this case, type='file' satisfies the requirement.
* The enctype in the form tag must be set to 'multipart/form-data'.
* The method in the form tag is a POST.

The request header above sets the content type correctly: Content-Type: multipart/form-data.

The two options that have no content disposition are incorrect. The content disposition needs to be available and show the file name:

Content-Disposition: form-data; name="myFile"; filename="C:\Testing\test.txt"

The option that sets the content type to 'x-www-form-urlencoded' is incorrect. This content type is the default when one is not provided, but it is the wrong one for uploading files.

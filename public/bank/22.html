In the following HTML page, a function stepThroughIt() is defined:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           function stepThroughIt() {
                try {
                     try {
                          throw new Error('Division by zero');
                     }
                     catch (ex) {
                          console.error(ex.message);
                          throw ex;
                     }
                     finally {
                          console.log('Inner finally');
                          return;
                     }
                }
                catch (ex) {
                     console.error(ex.message);
                }
                finally {
                     console.log('Outer finally');
                }
           }

           stepThroughIt();
      </script>
</body>
</html>

What will get written to the log when the page is loaded in the browser?

Division by zero Outer finally
Division by zero Inner finally Outer finally
Division by zero Inner finally Division by zero Outer finally
Division by zero Inner finally

Explanation

The following will be written to the log:

Division by zero
Inner finally
Outer finally

The function stepThroughIt() is composed of nested try/catch blocks. The first try throws an error immediately. This error is handled in the catch block. In this case, only the ex.message is written to the console. Therefore, "Division by zero" is written to the console. In the case where only ex is logged (console.log(ex)), the following will get displayed instead: "Error: Division by zero".

After the error is logged, it is thrown again. The finally block needs to execute first, and "Inner finally" is written to the console. However, what is unique in this scenario is that the finally block has the return; statement. This return statement overwrites the previous throw. In other words, the inner throw will not get executed, and therefore the outer catch will not run (another "Division by zero" will not be written to the console). Since finally statements are always executed, the "Outer finally" will be written to the console.

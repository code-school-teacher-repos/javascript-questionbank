Interact with HTML Forms
SubObjective:

Complete and debug code that retrieves input from forms and sets form field values

You are creating a form that interacts with different types of users. You investigate the available HTML widgets and decide to use some of them.

Which widget correctly implements the associated description?

The user can select the logo color from the color dialog. <p>      <input type="picker" name="logo" id="logo"> </p>
The user can select one or more images from the image file dialog. <p>      <input type="file" name="imageFile" id="imageFile" accept="image/*" multiple> </p>
The user can select a story estimate between 1 and 21 in an increment of two. <p>      <input type="text" name="storyEstimation" id="storyEstimation" min="1" max="21" step="2"> </p>
The user is able to use a slider to estimate the number of party goers, which should range from 10 to 200. <p>      <input type="range" name="partyCount" id="partyCount" step="5"> </p>

Explanation

The ability of the user to select one or more images from the image file dialog is implemented correctly. The type is "file", the "accept" is "image/*", and it can accept multiple images:

<p>
     <input type="file" name="imageFile" id="imageFile" accept="image/*" multiple>
</p>

The option to select a story estimate between 1 and 21 in an increment of two is incorrect. The type should be "number", not "text".

<p>
     <input type="number" name="storyEstimation" id="storyEstimation" min="1" max="21" step="2">
</p>

The option given to the user to use a slider to estimate the number of party goers, which should range from 10 to 200, is lacking the minimum and maximum values. The correct syntax is:

<p>
     <input type="range" name="partyCount" id="partyCount" min="10" max="200" step="5">
</p>

In order for the user to select the logo color from the color dialog, the type should be "color", not "picker":

<p>
     <input type="color" name="logo" id="logo">
</p>

Program with Variables, Data Types, and Functions
SubObjective:

Declare and use variables of primitive data types

You have the following HTML page:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <div id="result"></div>
      <script>
           function doOperation() {
                document.getElementById("result").innerHTML = languageName;
                languageName = "JavaScript";
                var languageName;
                var fullName = ' ';
                if (languageName === 'JavaScript')
                     fullName = "Intro to " + languageName
                else
                     fullName = "Class Not Created Yet!";

                console.log(fullName);
           }

           doOperation();
      </script>
</body>
</html>

What will be displayed on the screen when the page is loaded in the browser?

Intro to JavaScript
null
undefined
Nothing because an error will be displayed in the console instead

Explanation

"undefined" will be displayed on the page.One important item to notice in the function declaration is that the variable languageName is used before it is declared. So the question is: What will be the content of the variable? And, will this throw an error?

First of all, the variable is eventually declared in the function, so this will not cause an error. The variable will be undefined instead. This relates to an important concept called Hoisting. To understand this concept, keep in mind that the compiler will take a first walk through the code. With this walk it will be able to determine what variables and functions are available. In this first walk, the variables are not assigned to their content yet. Therefore, the system knows that there is a variable called "languageName". In memory, this variable is moved, or hoisted, to the top of the function. So the beginning of the function looks like this:

var languageName;

With such variable declaration, the value is undefined. So when the following statement tries to assign the variable to the result's inner HTML, the display will show "undefined":

document.getElementById("result").innerHTML = languageName;

The content will not show "null". In addition, the question asks about what gets displayed in the page, not in the console. When the function executes, "Intro to JavaScript" will display in the console, but "undefined" will be displayed in the page.

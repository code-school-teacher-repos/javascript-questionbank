You are redesigning the form for the enrollment section of a college website. One of the elements on the page is the SAT score.

You need to choose the best input type to use for this element.

What is the BEST input type to use?

text
range
number
select

Explanation

The best fit for this type of element is range. Here is an example:

<div>
   <label for="sat">
      Enter the SAT score:
      </label>
      0
      <input type="range" id="sat" name="sat" value="800" min="0" max="1600" step="10">
      1600
</div>

The range input type is made of a horizontal slider bar with the minimum value on the left and the maximum value on the right (not all browsers support showing the min and the max in the control. If they are not supported, you can add them in the HTML). The user can swipe a vertical handle bar left/right to decrease/increase the value. When you use the range input type, the user can select the value from a list, and therefore the chance of an error is very limited. You are also able to set the minimum and maximum values allowed, as well as a step. Many users might not be able to remember the exact score, so stepping by 10 in this scenario can be helpful. All recent browsers are almost up-to-date with this type of element. It is also visually simple for the user to move the handle on the control and select the score.

You should not use the number input type. The number input type is composed of an input box with up and down arrows on the right-hand corner. The user can click the arrows to increase/decrease the value. Using a number type is also doable, but not as good as the slider (range). The range is visually more appealing, and with a number it is harder for the user to click the up and down arrows to go between large brackets (for example the default value is 600 and the user needs to go up to 1450).

You should not use the select input type. You would need to provide all possible values from 0 to 1600.

You should not use the test input type. You would need to read the value, convert to number, make sure it is between the minimum and maximum allowable values, and generate error messages when the number is invalid. This is too much work compared to a type that already supports this feature (range).

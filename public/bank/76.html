Interact with the Document Object Model
SubObjective:

Identify and construct the Document Object Model (DOM) tree

You are creating a page in an application that contains a grid of records.

When a user clicks on one record, the background color changes to yellow. If the user exits the application completely and comes back to it later in the day, you want the application to remember the record and highlight it.

You need to add the code to accomplish this task.

Which feature should you use?

Session storage
Local storage
Global variables
Location object

Explanation

You need to use local storage to accomplish this task. You can use local storage or session storage to save data as the user is browsing the application. However, with session storage, the data is maintained as long as the session is still active (the user did not close the window or the tab).

For this scenario, the user exits the application and comes back to it later in the day. Session storage will be wiped out, but not local storage. With local storage, the data has no expiration time. Here is an example how you might be able to use the local storage:

localStorage.setItem("rowSelected", "15");
....
var selection = locaStorage.getItem("rowSelected");

Global variables will not work in this case because they will also be wiped out as the user exits the application (closes the window).

The location object gets and sets the current URL for the window object. It is not applicable to the scenario of storing data.

Interact with HTML Forms
SubObjective:

Describe the form submission process

You have the following form:

<form method="post">
      <div>
           <label for="first">First Name:</label>
           <input type="text" id="first" name="firstName">
      </div>
      <div>
           <label for="Id">Last Name:</label>
           <input type="text" id="last" name="lastName">
      </div>
      <div>
           <input type="submit" value="Submit">
      </div>
</form>

A user fills out the form and submits it to the server.

What will the request body contain?

The HTTP request body is empty.
POST http://localhost:35132/Account/FormTest HTTP/1.1 Accept-Language: en-US Content-Type: multipart/form-data Content-Length: 27 Host: localhost:35132 firstName=John&lastName=Doe
POST http://localhost:35132/Account/FormTest?firstName=John&lastName=Doe HTTP/1.1 Accept-Language: en-US Content-Type: application/x-www-form-urlencoded Content-Length: 27 Host: localhost:35132
POST http://localhost:35132/Account/FormTest HTTP/1.1 Accept-Language: en-US Content-Type: application/x-www-form-urlencoded Content-Length: 27 Host: localhost:35132 firstName=John&lastName=Doe

Explanation

The request body will be as follows:

POST http://localhost:35132/Account/FormTest HTTP/1.1
Accept-Language: en-US
Content-Type: application/x-www-form-urlencoded
Content-Length: 27
Host: localhost:35132

firstName=John&lastName=Doe

This form is submitted using the "POST" method. That means the data is sent in the body, not in the URL. The first part of the request sets some properties such as the content type. The second part contains the data:

firstName=John&lastName=Doe

The option that states that the body is empty is incorrect. This is a POST request, and the data will be sent in the body.

The option that attaches the data to the URL is incorrect. Data is attached to the URL in "GET" requests.

The option that sets the content type to "multipart/form-data" is incorrect. The page did not specify the 'enctype' in the form. The default value is: application/x-www-form-urlencoded.

SubObjective:

Complete and debug loops

You load the following page in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           var statements = ["There are many ways to handle the same problem",
                                         "You can declare variables using let and var",
                                         "The apples are in the basket",
                                         "Did you put the groceries in the fridge?",
                                         " "];
           var j = 0, k = 0, m = 0;
           handleThem();
           console.log(`j:${j},k:${k},m:${m}`)

           function handleThem() {
                statements.forEach(function (item) {
                     workOnIt(item);
                });
           }

           function workOnIt(item) {
                var result = item.split(" ");
                var i = 0;
                for (; i < result.length;) {
                     switch (result[i].toLowerCase()) {
                          case "in":
                               j++;
                               break;
                          case "the":
                               k++;
                               break;
                          default:
                               m++;
                     }
                     i++;
                }
           }
      </script>
</body>
</html>

What will be displayed in the console?

j:3,k:7,m:22
j:2,k:3,m:28
j:2,k:5,m:25
j:4,k:6,m:19

Explanation

The following will be displayed in the console:

j:2,k:5,m:25

The statements array contains a list of strings. The function handleThem() loops over each statement and passes it to the workOnIt() function, which splits the statement using spaces and loops over the array of words.

The for loop uses a switch statement to look at each word and decide what to do about it. The two main cases are for "in" and "the". The j variable accumulates the occurrences of "in", and the k variable accumulates the occurrences of "the". Everything else defaults to m.

Each case block uses the break statement appropriately so execution will not fall erroneously to the case below it.

By the time the loop is over, j will have 2, k will have 5 and m will have 25.

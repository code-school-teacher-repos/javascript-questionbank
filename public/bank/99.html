Interact with HTML Forms
SubObjective:

Complete and debug code that retrieves input from forms and sets form field values

You are creating a student form that needs to be submitted to the server. One of the elements on the form is the course ID.

You need to create the course ID to meet the following requirements:

* When then user clicks on the label, the input box gets the focus.
* The input box shows the format to be used when typing the course ID.

Which code segment should you use?

<div>      <label for="courseId">Course Id:</label>      <input type="text" id="Id" name="courseId" value="CSC-100"> </div>
<div>      <label for="courseId">Course Id:</label>      <input type="text" id="Id" name="courseId" placeholder="CSC-100"> </div>
<div>      <label>Course Id:</label>      <input type="text" id="Id" name="courseId" value="CSC-100"> </div>
<div>      <label for="Id">Course Id:</label>      <input type="text" id="Id" name="courseId" placeholder="CSC-100"> </div>

Explanation

You should use the following code segment to meet the requirements:

<div>
      <label for="Id">Course Id:</label>
      <input type="text" id="Id" name="courseId" placeholder="CSC-100">
</div>

The label preceding the input box needs to have the for attribute that references the ID of the input. Because the id is Id, the label should be: <label for="Id">. Now when the user clicks on the label, the input box will have the focus. Using the for attribute to refer to the name of the input box instead of the id is incorrect.

The second requirement states that the format of the course id needs to show in the input box. This is accomplished by using the placeholder attribute. The format will appear as soon as the form loads, but disappears when the user starts typing. The input box will look like the following:

<input type="text" id="Id" name="courseId" placeholder="CSC-100">

Using the value attribute instead of placeholder is incorrect because that will put the actual text in the input box, and that is not what the requirement states.

Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug a function that accepts parameters and returns a value

The script block of an HTML page contains the following:

<script>
      var letters = ['A', 'M', 'F', 'G', 'U', 'T', 'R', 'W', 'Q', 'D'];

      function getLetter(index) {
           if (index < 0) return;
           return letters[index];
      }

      function getLetter(index, pushIt) {
           if (pushIt)
                letters.push('P');
           return letters[index];
      }

      function getLetter(index, sortIt, shiftIt) {
           if (index < 0) return;
           if (shiftIt)
                letters.shift();
           if (sortIt)
                letters.sort();
           return letters[index];
      }

      console.log(getLetter(5, true));
</script>

Which letter will be displayed in the console?

U
T
Q
R

Explanation

The letter that will be displayed in the console is Q.

The script contains three different declarations for the function getLetter(). Functions in JavaScript are function objects. When functions are declared with the same name in the same scope, the last one loaded is the active one. In other words, the others are overwritten. So as far as JavaScript is concerned, there is only one function named getLetter(), and it is the one with the parameters index, sortIt, and shiftIt, since this is the last one loaded.

When calling a function with too many parameters, the additional ones are ignored. When calling it with too few parameters, the extra ones are assigned to undefined.

Based on that, the parameter shiftIt is undefined, the index is 5, and the sortIt parameter is true.

The array is then sorted, and the value at index 5 is returned (keeping in mind that JavaScript array indexing starts at 0). After sorting the array, the letter at location 5 is Q.

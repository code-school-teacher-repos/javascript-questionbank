You are working on the form shown in the answer area. The form will have an input box so the user can enter a contact phone number.

You need to design the pattern for the input.

Which Regular Expression pattern should you use? To answer, select the appropriate option from the drop-down menu.

Explanation

The correct pattern for a telephone number is:

^\d{10}$|^(\(\d{3}\)\s*)?\d{3}[\s-]?\d{4}$

This pattern allows the user to enter, in addition to the numbers, parentheses, spaces or dashes.

The pattern .{10,} is invalid because it only means that the user should enter 10 or more characters. It does not account for dashes and parentheses or an upper bound of characters.

The following option is more of a pattern for a password. It requires at least one number and one uppercase and lower case letter, and 10 or more characters:

(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{10,}

This following option matches an email address (someone@somewhere.domain): characters, then @, then more characters, then dot, and then domain:

[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,3}$

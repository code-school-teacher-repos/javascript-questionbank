You are adjusting a web application to support 5 or 9 digit postal codes. You are working on the form shown in the answer area.

You need to design the pattern for the input.

Which pattern should you use? To answer, select the appropriate option from the drop-down menu.

Explanation

The correct pattern to use for the 5 or 9 digit zip code support is:

^\d{5}(-\d{4})?$

This pattern starts with 5 digits followed by an optional section (surrounded by parentheses). That section contains a dash followed by 4 digits. In other words, the user can enter 5 digits, or 5 followed by a dash and 4 more digits.

The pattern ^\d{5}(\d{4})?$ is not valid because it ignores the dash between the 5 and 4 digits.

The pattern ^\d{5}-\d{4}$ is not valid either because it removes the parentheses and question mark, making the dash and 4 additional digits required. In other words, the user does not have the option to only enter 5 digit zip codes.

The pattern ^\d{5}[ -][0-9]$ is incorrect. It supports the 5 digits, but then it requires a space or a dash followed by only one digit between 0 and 9.

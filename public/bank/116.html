Interact with HTML Forms
SubObjective:

Describe the form submission process

You are designing a form for an Enrollment page to work with IE 6. When the form is completed, it should be submitted to the same page.

You need to design a form with the appropriate action.

Which form element should you use?

<form>
<form action="/Employee">
<form action="http://some-company.com">
<form action="#">

Explanation

The form with the correct action is: <form action="#">. You use the action attribute to specify where the data will be sent. Because the users will be using an older browser, the action attribute is required. Because the form is submitted to the same page, a "#" symbol is used.

The <form> without the action attribute is the right way to go when submitting to the same page using newer browsers that support HTML 5. With that support, the action attribute is no longer required.

The other two forms are submitting using an absolute URL (<action=http://some-company.com>) and a relative one (<action="/Employee">). They do not submit to the same Enrollment page as required.

Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug a function that accepts parameters and returns a value

You have the following code:

function applyIt(x, ...elements) {
return elements.map(e => Math.pow(e, x));
}
console.log(applyIt(2, 1, 2, 3, 4, 5));

You add the code to a script tag and load the page in the browser.

What will be the result?

Array [ 4,1, 4, 9, 16, 25 ] will be displayed in the console.
A syntax error will be caused by "=>".
A syntax error will be caused by "...".
Array [ 1, 4, 9, 16, 25 ] will be displayed in the console.

Explanation

The code will cause the following to show in the console:

Array [ 1, 4, 9, 16, 25 ]

The rest parameter (...) and the arrow function expression (=>) are part of the language and will not cause syntax errors. The rest parameter in the function declaration represents an infinite number of arguments as an array. The arrow function (also known as fat arrow function) is a short representation of a function. Anything on the left hand side of the arrow represents the input parameters, and everything on the right is the function body.

In the code above, the applyIt() function takes x as a first parameter, then the rest parameters as an array. The body uses the map() function of the array to map each element based on the arrow function. That means that every element in the array is raised to the power of x. Since x is 2, every element is squared. The array is made of every item after x. That means:

1, 2, 3, 4, 5 => 1, 4, 9, 16, 25

This result is displayed in the console.

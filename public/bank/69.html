Implement and Analyze Decisions and Loops
SubObjective:

Complete and debug decision statements

You load the following page:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           var functionKeys = ["F1", "F2", "F5", "F3", "F7", "F6", "F9", "F8"];
           var result = ' ';

           function func1(key) {
                switch (key) {
                     case "F1":
                          result += "F1";
                     case "F2":
                          result += "F2";
                     case "F3":
                          result += "F3";
                     case "F4":
                          result += "F4";
                     case "F5":
                          result += "F5";
                }
           }

           functionKeys.forEach(function (k) { func1(k) });
           console.log(result);
      </script>
</body>
</html>

What will result contain?

F1F2F3F4F5F2F3F4F5F5F3F4F5
F1F2F3F5F2F3F5F5F3F5
F1F2F3F5
F1F2F5F3

Explanation

When the page is loaded in the browser, result will contain the following:

F1F2F3F4F5F2F3F4F5F5F3F4F5

The array of function keys is declared and the forEach loops over these keys and passes each key to func1().

The function uses a switch statement and handles the case of each key. For each key, the value is added to the result variable. However, it is important to note that the case statements are missing their corresponding break statements. Because of that, whenever a case is handled, the code falls through to the cases under it and their values are added to the result as well. For example, for "F3", "F3" is added to the result, but so are "F4" and "F5".

Going through each key in the array and matching their values with the cases in the switch statement will accumulate "F1F2F3F4F5F2F3F4F5F5F3F4F5" in result.

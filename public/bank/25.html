SubObjective:

Complete and debug code that interacts with the Browser Object Model (BOM)

You load the following HTML page in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
      <meta charset="utf-8" />
</head>
<body>
      <div id="name">Tester</div>
      <script>
           function setStorageItems() {
                localStorage.setItem('storage.name', document.getElementById('name').innerHTML);
                sessionStorage.setItem('storage.color', 'yellow');
                localStorage.setItem('storage.language', 'JavaScript');
                sessionStorage.setItem('storage.page', 'hello.html');
           }

           setStorageItems();

           function getStorageItems() {
                var name = localStorage.getItem('storage.name');
                var color = localStorage.getItem('storage.color');
                var language = localStorage.getItem('storage.language');
                var page = sessionStorage.getItem('storage.page');

                console.log(name);
                console.log(color);
                console.log(language);
                console.log(page);
           }

           getStorageItems()
      </script>
</body>
</html>

Then you comment out the call setStorageItems(), close all browser windows, and load the page again in the browser.

What will be displayed in the console?

Tester null JavaScript hello.html
null Tester null JavaScript
Tester null JavaScript null
undefined Tester undefined JavaScript

Explanation

The output in the console will be:

Tester
null
JavaScript
null

Note the difference between localStorage and sessionStorage. With localStorage, the data remains when the browser is closed and reopened. In the case of sessionStorage, the data remains as long as the window or the tab is open (the data survives for that user's session).

In this scenario, you store different data using both storage mechanisms:

localStorage.setItem('storage.name', document.getElementById('name').innerHTML);
sessionStorage.setItem('storage.color', 'yellow');
localStorage.setItem('storage.language', 'JavaScript');
sessionStorage.setItem('storage.page', 'hello.html');

The data is retrieved as follows:

var name = localStorage.getItem('storage.name');
var color = localStorage.getItem('storage.color');
var language = localStorage.getItem('storage.language');
var page = sessionStorage.getItem('storage.page');

Then the call to setStorageItems() is commented out, all windows are closed, and the page is loaded again. That means all session storage data will be lost. In addition, the color is set using sessionStorage, but retrieved using localStorage.

It is also important to note that when a value is not available, a result of null will show, not undefined.

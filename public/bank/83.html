Interact with the Document Object Model
SubObjective:

Identify and handle HTML events

You load the following page in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <div onclick="takeAction()">
           <div onclick="takeAction()">
                <p onclick="takeAction()">There are many variations of passages of Lorem Ipsum available.</p>
           </div>
      </div>
      <div id="result"></div>
      <script>
           var count = 1;
           function takeAction() {
                document.getElementById("result").innerHTML = count++;
           }
      </script>
</body>
</html>

You click on the text in the paragraph three times.

What gets displayed in the div with id result after the consecutive clicks?

2 4 6
3 6 9
1 1 1
1 2 3

Explanation

The following will be displayed in the div with id result after the consecutive clicks:

3 6 9

The paragraph has an onclick() event that calls the takeAction() function when the user clicks on it. Also, there are handlers for the onclick events in the parent divs. Since the onclick event propagates (moves up in the hierarchy), takeAction() will be called for the outer divs as well. The function increments the variable count at every call. That results in 3, 6, and 9 being displayed in the div with id result when the user clicks three times on the paragraph.

It is important to note that the event has a function called stopPropagation(). As the name implies, if you call this function inside the takeAction() function, the event will not propagate to the parent divs. In that case, 1, 2, and 3 would be displayed instead. In order to call stopPropagation(), you need to pass the event as a parameter to takeAction() in the HTML and the JavaScript implementation.

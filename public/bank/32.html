Program with Variables, Data Types, and Functions
SubObjective:

Declare and use variables of primitive data types

A date is declared as follows:

var myDate = new Date(2018, 5, 8, 7, 30, 25);

When printed in the browser window, you need it to show in the following format:

2018-06-08T11:30:25.000Z

Which call should you use?

myDate.toLocaleString()
myDate.toGMTString()
myDate.toDateString()
myDate.toISOString()

Explanation

The call that meets the requirement is: myDate.toISOString(). JavaScript supports the Date object, which is the number of milliseconds since January 1, 1970 UTC. A date can be constructed using different options. Passing no arguments to Date() represents the current date. A user can also pass the number of milliseconds since 1970 or a string representing a specific date. Also, the user has the options of passing the fields one by one representing year, month, date, hour, minute, second, and millisecond.

In this scenario, a Date is created using the final approach above:

var myDate = new Date(2018, 5, 8, 7, 30, 25);

There are different functions for getting and setting the date pieces, such as getTime(), getMonth(), setHours() and setSeconds(). There are also functions for converting to different date formats. For this example, to convert the date to 2018-06-08T11:30:25.000Z, the function to call is myDate.toISOString().

The other options work differently:

myDate.toDateString() = Fri Jun 08 2018
myDate.toGMTString() = Fri, 08 Jun 2018 11:30:25 GMT
myDate.toLocaleString() = 6/8/2018, 7:30:25 AM

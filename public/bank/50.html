SubObjective:

Complete and debug code that uses built-in Math functions

You are using the Math object to create a package for financial calculations.

For each of the following statements, select Yes if the statement is true. Otherwise, select No.

Explanation

Math.abs(x) when x is 1024 evaluates to 1024, not -1024. The abs() method returns the absolute value of a number (the number without the negative sign). For example, Math.abs(-1024) is 1024 and Math.abs(1024) is 1024 as well.

All the methods in the Math object are static. Since all the methods and properties in the Math class are static, a Math object is not created to call these properties and methods. For example, the following is incorrect:

var m = new Math()
var x = m.abs(-34);

Math.ceil(0.5) evaluates to 1. The ceil() method returns the smallest integer greater than or equal to the given number. In this scenario, the smallest integer that is greater than or equal to 0.5 is 1. Other examples include Math.ceil(3.3) is 4, and Math.ceil(87.1) is 88.

Math.acos(0.5)*(180/Math.PI) evaluates to 60. The Math.acos() method returns the arccosine of a given number. The parameter needs to be between -1 and 1, and the result is between 0 and PI (3.14159). That is the result in radians, not degrees. In order to convert the answer to degrees, it needs to be multiplied by (180/Math.PI).

The same concept applies to the Math.asin() method, which returns the arcsine of a given number. In this scenario,

Math.acos(0.5) evaluates to 1.047.
Math.asin(0.5) evaluates to 0.524.

After multiplying by (180/Math.PI), the result is:

Math.acos(0.5)*(180/Math.PI) evaluates to 60.
Math.asin(0.5)*(180/Math.PI) evaluates to 30.

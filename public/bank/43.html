Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug code that uses objects

A date is created using the following statement:

var d = new Date(2017,11,17,8,53,9);

Which three statements are valid regarding this date? Each correct answer presents a complete solution.

Time: 1513518789000 represents the number of seconds since January 1, 1900.
d.getDate() returns 17.
d.getDay() returns 0 to represent Monday.
Calling d.toLocaleTimeString() will result in 8:53:09 AM.
Calling d.toLocaleDateString() will result in 12/17/2017.

Explanation

The following statements are true about the date object:

Calling d.toLocaleDateString() will result in 12/17/2017
Calling d.toLocaleTimeString() will result in 8:53:09 AM
d.getDate() returns 17

There are four different ways to construct a Date object. The one used in this question is constructed using year, month, date, hours, minutes, seconds. Since the month starts at 0 (January), the value 11 is for December, not November.

The locale time string of 8:53:09 AM is correct as specified in the Date constructor since the last three parameters (8, 53, and 9) represent the hours, minutes, and seconds respectively. By calling d.toLocalTimeString(), the time is formatted properly as 8:53:09 AM.

The getDate() function returns the day of the month (1 - 31). The value 17 is correct based on the constructor new Date(2017,11,17,8,53,9). The third parameter, 17, corresponds to the day of the month returned by getDate().

The time: 1513518789000 represents the number of seconds, but that will be since January 1, 1970, not since January 1, 1900.

The call to d.getDay() returns 0. It returns the day of the week (0-6). 0 is Sunday, not Monday.

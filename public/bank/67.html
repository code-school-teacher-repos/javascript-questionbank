Implement and Analyze Decisions and Loops
SubObjective:

Complete and debug decision statements

The following page is loaded in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           var elements = ["one", "two", "three", "seven", "six", "five"];
           handleElements(elements);

           function handleElements(e) {
                e.sort();
                if (e[0] !== "five")
                     console.log("Not found!");
                     console.log("It was not the first element!");
                e.push("nine");
                e.shift();
                if (e[0] !== "five")
                     console.log("Lost it!");
                if (e.length === 6)
                     console.log("But I still have six elements!")
                let index = 0;
                e.forEach(function (item) {
                     if (item.length === 3)
                          index++
                });
                if (index === 3)
                     console.log("Got the ones I was looking for!")
                }
      </script>
</body>
</html>

What will be displayed in the console?

Lost it! Got the ones I was looking for!
It was not the first element! Lost it! But I still have six elements! Got the ones I was looking for!
Lost it! But I still have six elements! Got the ones I was looking for!
Not found! It was not the first element! Lost it! But I still have six elements! Got the ones I was looking for!

Explanation

The following will display in the console:

It was not the first element!
Lost it!
But I still have six elements!
Got the ones I was looking for!

The script creates an array and passes it to handleElements(). The function applies some operations on the array and does some condition checks along the way that print statements to the console when the condition evaluates to true.

The first action is to sort the array. That places "five" as the first element. However, the if block is missing the curly braces ({}), which means that only the first console.log applies to the condition. The statement "It was not the first element!" prints.

After pushing "nine" to the end, and shifting (removing the first element), "five" is no longer the first. That causes "Lost it!" to display. But the array still contains six elements. That also makes "But I still have six elements!" show on the console.

The forEach loop visits each element in the array. If the length of the element is three, the index is incremented. There are three items that satisfy that condition: one, two, and six.

Since the index is 3, "Got the ones I was looking for!" also shows in the console.

Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug code that uses objects

An HTML page loads with the following script:

<script>

      var course = {
           name: 'Data Analysis and Design',
           id: 10005,
           semester: 'Spring',
           students: {
                student1: {
                     name: 'James',
                     totalCredits: 10
                },
                student2: {
                     name: 'Jessica',
                     totalCredits: 8
                }
           }
      }

      function findIt(obj, objName, prop) {
           if (obj.hasOwnProperty(prop)) {
                console.log(`${objName}.${prop} is: ${obj[prop]}`);
           }
      }

      findIt(course, 'Course', 'id');
      findIt(course, 'Course', 'name');
      findIt(course, 'Course', 'students');
</script>

What will be displayed in the console?

Course.id is: 10005 Course.name is: Data Analysis and Design
Course.id is: 10005 Course.name is: Data Analysis and Design Course.students is: [{name: 'James',totalCredits: 10}, {name: 'Jessica',totalCredits: 8}]
Course.id is: 10005 Course.name is: Data Analysis and Design Course.students is: {student1, student2}
Course.id is: 10005 Course.name is: Data Analysis and Design Course.students is: [object Object]

Explanation

The following will be displayed in the console:

Course.id is: 10005
Course.name is: Data Analysis and Design
Course.students is: [object Object]

The object course has some properties and an inner object named students that has properties of its own. The function findIt()is passed the object and a property to check whether the object contains that property. This is accomplished using hasOwnProperty(). As the name implies, this method returns a Boolean to indicate whether the object has the property as its own (not inherited).

In the first two calls to the function, id and name are properties of course. Their content will be logged. In the third call, however, students is another object and its content cannot be logged. So, [object Object] will be displayed. The log statement will not be able to display {student1, student2} or [{name: 'James',totalCredits: 10}, {name: 'Jessica',totalCredits: 8}].

If you need to check/display properties of objects and their inner objects, you can build a function that calls itself recursively while passing the inner objects as parameters.

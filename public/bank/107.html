Interact with HTML Forms
SubObjective:

Complete and debug code to perform input validation

You are working on a personal information form for Human Resources.

For each of the following statements, select Yes if the statement is true. Otherwise, select No.

Explanation

The following statements are true:

First name and last name are required fields.
The phone number input field should have a pattern attribute.
The field for rating the overall company website should be a range slider.
The Contact Me email address should be an input box of type email.

Necessary information should always be made required. The user should not be able to submit the form when critical information is not yet provided. First name and last name match this category and they should be provided before submitting the form.

The phone number is an input box of type 'tel' and it should have the 'pattern' attribute. With such attribute, the developer can clearly indicate the format of the telephone number in order for it to be considered valid (for example, the pattern tries to match (xxx)xxx-xxxx).

The range sliders are an excellent way of asking the user for numeric input between specific brackets. For example, with the type 'range' input box, you can specific the min, max, and step values. The user can then, move the handle between the min and max to indicate the value. If you use the input box of type "text", then you will have to convert the value to numeric after reading it and handle all the validations.

Having the email address input box of type 'email' will help because the user will not be able to submit the form until a valid email is provided. This field should also be made required.

Many UX (User eXperience) designers agree that it is not a good programming practice to include a reset button that clears out all the fields on the form. The reason is that this field sits very close to the submit button and the user might click it by mistake. In addition, if the user makes mistakes in filling out the fields, error handling and appropriate input types should handle the cases so the user does not have to clear the fields and start over.

The start date field should be of type 'date', not 'text'. Many browsers support the date dropdown for the user to select from. You can also indicate min and max values so the user will not be able to enter values out of range.

Server validation should still exist in addition to the client side validation. Hackers can intercept the data on the way to the server and enter malicious information.

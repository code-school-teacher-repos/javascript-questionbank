An HTML page references two JavaScript files as follows:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
      <style>
           .format-output {
                width: 300px;
                height: 100px;
                border: 1px solid blue;
                font:bold 20px "courier new";
                text-align:center;
           }
           </style>
           <script src="Scripts/Common1.js"></script>
           <script src="Scripts/Common2.js"></script>
</head>
<body>
      <div id="result" class="format-output"></div>
      <script>
           var data1 = "Action";
           var data2 = "This is a test about using JavaScript";
           var result = " ";
           if (checkInput(data1))
                result += doAction(data1);
           if (checkInput(data2))
                result += doAction(data2);
           document.getElementById("result").innerHTML = result;
      </script>
</body>
</html>

Common1.js


function checkInput(field) {
      if (!field)
           return false;

      const MAX = 20;
      return field.length <= 20;
}

Common2.js


function doAction(data) {
      if (!data)
           return ' ';
      var newData = ' ';
      for (var i = data.length - 1; i >= 0; i--) {
           newData += data[i];
      }
      return newData;
}

What will be displayed in the browser when the HTML page is loaded?

Explanation

The following will be displayed in the browser:

noitcA

In the code above, the HTML page is referencing two JavaScript files: Common1.js and Common2.js. These files are referenced using the script tag and the src attribute. This attribute references the files in the Scripts folder. When the HTML page is loaded, these files are retrieved as well. In most cases, the browser can cache these files locally so that if they are referenced again, they can be served from the browser cache instead of making additional calls to retrieve them.

Each file has one function. In the first file, checkInput() is checking whether the data passed as a parameter is less than or equal to 20 characters in length. In the second file, the doAction() is reversing the content of the data by looping backwards through the characters and building and returning new set of data.

In the script section of the HTML page, there is a check made before reversing the content of the data. data1 passes the validation and its content is reversed. data2, however, does not. For that reason, only data1 is reversed and the content of the result is "noitcA".

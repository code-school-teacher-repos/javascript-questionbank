Implement exception handling

The body of the HTML page contains the following script:

<script>
      var fileInfo = 'Could not process the file';
      var reportInfo = 'The daily report was not generated';
      var invoiceInfo = 'The invoices were not created';
      try {
           logInfo(fileInfo, reportInfo, invoiceInfo);
      }
      catch (e) {
           console.log('Could not call the function to log the info!');
      }
      finally {
           console.log('Executed the finally block');
      }

      function logInfo(info) {
           var date = new Date();
           console.log(`INFO: ${info}; LOGGED ON ${date}`);
      }
</script>

What will be displayed in the console when this page is previewed in the browser?

INFO: Could not process the file; LOGGED ON Fri Nov 03 2018 20:19:53 GMT-0400 (Eastern Standard Time) Could not call the function to log the info! Executed the finally block
Could not call the function to log the info! Executed the finally block
INFO: Could not process the file; LOGGED ON Fri Nov 03 2018 20:19:53 GMT-0400 (Eastern Standard Time) Executed the finally block
Could not call the function to log the info!

Explanation

The following will be displayed in the console:

INFO: Could not process the file; LOGGED ON Fri Nov 03 2018 20:19:53 GMT-0400 (Eastern Standard Time)
Executed the finally block

The script starts by creating three variables with different strings. Then, a call is made to the logInfo() function in the try block. The three variables are passed to the function as parameters. The function creates a date objects and logs the information to the console. There are two key elements to note with this set of code:

1. The function takes one parameter, and yet it is called with three.
2. The function is called inside a try/catch/finally block.

When it comes to the function call, JavaScript will still call the function by taking the first parameter and ignoring the other two. Therefore, logInfo() is still called with fileInfo as a parameter without causing any errors or exceptions.

The try/catch/finally blocks are used to wrap code that might cause errors/exceptions. The code is placed inside the try block. The catch blocks handle the appropriate error caused by the code in the try block. The finally block will always execute whether there were errors or not. This block is usually used for cleanup, such as when there are file handles to close.

In this scenario, the call to the function succeeds for the reason discussed above, and something similar to the following will be printed in the console. (The date will differ because it reflects the current date.):

INFO: Could not process the file; LOGGED ON Fri Nov 03 2018 20:19:53 GMT-0400 (Eastern Standard Time)

The catch block will not execute because there were no errors. Then the finally blocks executes and the following is printed in the console:

Executed the finally block

Therefore the total output will look like the following:

INFO: Could not process the file; LOGGED ON Fri Nov 03 2018 20:19:53 GMT-0400 (Eastern Standard Time)
Executed the finally block

Program with Variables, Data Types, and Functions
SubObjective:

Declare and use variables of primitive data types

The following code snippet declares some variables and calls a function to print their types:

var x;
var y = true;
var z = 123;
var t = 34.65;
var u = 'JavaScript';
var v = { first: 'JavaScript', last: 'Tester' };
var w = new Date();
var m = null;
var n = undefined;

function findType() {
      var result = `type of x: ${typeof (x)}\n
           type of y: ${typeof (y)}\n
           type of z: ${typeof (z)}\n
           type of t: ${typeof (t)}\n
           type of u: ${typeof (u)}\n
           type of v: ${typeof (v)}\n
           type of w: ${typeof (w)}\n
           type of m: ${typeof (m)}\n
           type of n: ${typeof (n)}\n`

      alert(result);
}

findType();

Explanation

The correct types of the variables declared is as follows:

type of x: undefined
type of y: boolean
type of z: number
type of t: number
type of u: string
type of v: object
type of w: object
type of m: object
type of n: undefined

Variables in JavaScript can be declared using the var, let, or const keywords. Unlike other programming languages such as C# and Java, declaring a variable in JavaScript does not include the type. For example, to declare a variable that contains the course name, you use the following:

var courseName = "Intro to JavaScript";

JavaScript will "understand" that this is a string because of the initialization. But that does not stop the developer from placing another type in courseName. For instance, the following is allowed:

courseName = 2475;

In order to determine the type of the variable, the typeof operator is used. In this scenario, multiple variables are declared. Some of them are initialized, and others are not. When a variable is not initialized, its type is undefined. When initialized, JavaScript is able to determine the type of the variable. For example, setting y to true, will make y of type boolean. There are other exceptions, however, such as setting z to 123 and t to 34.65 will make them both numbers. Also, v is an object and w is an object as well (not a date).

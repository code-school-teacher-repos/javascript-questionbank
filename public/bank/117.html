Interact with HTML Forms
SubObjective:

Describe the form submission process

You have the following form:

<form method="get">
      <div>
           <label for="first">First Name:</label>
           <input type="text" id="first" name="firstName">
      </div>
      <div>
           <label for="Id">Last Name:</label>
           <input type="text" id="last" name="lastName">
      </div>
      <div>
           <input type="submit" value="Submit">
      </div>
</form>

A user fills out the form with 'John' in the first name and 'Doe' in the last name.

Which URL will the form submit?

http://localhost:35132/Account/FormTest
http://localhost:35132/Account/FormTest?firstName=John&lastName=Doe
http://localhost:35132/Account/FormTest?first=John&last=Doe
http://localhost:35132/Account/FormTest#first=John?last=Doe

Explanation

The form will submit the following URL:

http://localhost:35132/Account/FormTest?firstName=John&lastName=Doe

The form is using the "GET" method. With this method, the data is submitted to the server using the URL. There will be no data sent in the form body as it is the case with a "POST".

The format of the data will be: main URL followed by '?', then key-value pairs for the data joined with '&'. The main URL is http://localhost:35132/Account/FormTest. Joining the fields with '&' and attaching that to the URL using '?' will result in:

http://localhost:35132/Account/FormTest?firstName=John&lastName=Doe

The option that does not pass any data after the URL is incorrect because with a 'GET' method, the data is attached to the URL.

The option that uses '#' after the URL and '?' to join the key-value pairs is incorrect. Key-value pairs are joined with '&' and they are attached to the URL using '?'.

The option that uses the field ID instead of the field name is incorrect. They key-value pair is made up of the field name and the value.

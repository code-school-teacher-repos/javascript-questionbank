Explanation

You should use the following form:

<form method="post">
     <label for="team">Favorite Baseball team: </label>
     <input type="text" name="team" id="team" list="myFavorite">
     <datalist id="myFavorite">
     <option>Baltimore Orioles</option>
     <option>Arizona Diamondbacks</option>
     <option>Boston Red Sox</option>
     <option>Atlanta Braves</option>
     <option>Chicago White Sox</option>
     <option>Chicago Cubs</option>
     <option>Cleveland Indians</option>
     <option>Cincinnati Reds</option>
     <option>Houston Astros</option>
     <option>Los Angeles Dodgers</option>
     </datalist>
</form>

The autocomplete box accomplishes the task. It provides an input box and a datalist of options. The input box is tied to the datalist using the 'list' property. As soon as the user starts typing, the form displays a list of matching options. The datalist contains options similar to the select box.

You should not use the form that provides the options without the datalist. The list of options would not be displayed as the user starts typing.

You should not use the code set that wraps the options with a select instead of a datalist. Although select shows the list of teams in a dropdown, that list is not tied to the input box, and as the user types in the input box, the drop-down will not narrow down the list of options.

You should not use the code set that adds the 'multiple' attribute to the select box. The user can select more than one team from the list, but there is no relation to the input box. As the user is typing in the input box, the select box will not change to narrow down the options.

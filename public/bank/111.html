Interact with HTML Forms
SubObjective:

Complete and debug code to perform input validation

In a form, you need to create an input box that satisfies the following requirements:

* The field cannot be empty.
* The user is expected to provide the name of a programming language. Supported languages are: JavaScript, Python, Smalltalk, and Java.
* The first letter of the language is case insensitive.
* The label should have a star marking a required field. The user should see "This field is required" when hovering over the star.
* When there is no match to a supported language name, the text should turn red.

Which three pieces of code should you use in your solution?

<input type="text" id="lang" name="language" required pattern="JavaScript|Python|Smalltalk|Java">
input:focus:invalid {      color: red; }
<input type="text" id="lang" name="language" pattern="[A-Z][A-Za-z' -]+">
input#language:focus:invalid {      color: red; }
<label for="lang">This field is required*</label>
<input type="text" id="lang" name="language" required pattern="[Jj]avaScript|[Pp]ython|[Ss]malltalk|[Jj]ava">
<label for="lang">Select your favorite language      <abbr title="This field is required">*</abbr> </label>

Explanation

To match the requirements in the question, the complete solution should be as follows:

input:focus:invalid {
      color: red;
}

<div>
      <label for="lang">Select your favorite language
           <abbr title="This field is required">*</abbr>
      </label>
      <input type="text" id="lang" name="language" required
      pattern="[Jj]avaScript|[Pp]ython|[Ss]malltalk|[Jj]ava">
</div>

The input solution shown here meets the requirements. When the input has a focus and the user is entering invalid data, the color of the text will turn red. The input solution that indicates using the id (input#language:focus:invalid) is incorrect because the id of the field is 'lang', not 'language'.

The label surrounds an <abbr> tag that contains the star. The title of the <abbr> tag represents the text that should popup when the user hovers over the star.

Since the supported languages are JavaScript, Python, Smalltalk, and Java, you can use the pattern attribute to include these values specifically. In addition, you can include the first letter of the language in upper and lower case inside a square bracket, and that will make any of the two characters acceptable:

<input type="text" id="lang" name="language" required
pattern="[Jj]avaScript|[Pp]ython|[Ss]malltalk|[Jj]ava">

Using pattern="[A-Z][A-Za-z' -]+" is incorrect because it starts with an uppercase letter followed by any other letter, apostrophe, a space, or a dash, but there is no limit on the supported languages.

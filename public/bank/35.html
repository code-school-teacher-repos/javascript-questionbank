Program with Variables, Data Types, and Functions
SubObjective:

Declare and use variables of primitive data types

You need to convert undefined to different types.

Which conversion would be valid?

Conversion to a number will be NaN.
Conversion to an object will be {undefined: 'undefined'}
Conversion to a Boolean will be true.
Conversion to string will be empty string ("").

Explanation

Conversion to a number will be NaN. When an expression cannot be converted to a numeric type, as in the case of undefined, JavaScript will return NaN (not a number).

JavaScript is a type forgiving language. It will try its best to convert between the types to complete a specific operation. For example: "5" * "6" will be 30 since it will convert each item from string to number to get the result. In addition, "" will be converted to 0 when numeric and false when Boolean.

In this scenario, you are trying to convert undefined to object, string, number, and Boolean.

When converting undefined to string, it will be "undefined", not empty string ("").

In the case of a Boolean type, undefined is false. So are null and empty strings.

For object conversion, JavaScript cannot convert undefined to an object. For that conversion, it will throw a TypeError.

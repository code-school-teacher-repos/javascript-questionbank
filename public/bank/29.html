Program with Variables, Data Types, and Functions
SubObjective:

Declare and use variables of primitive data types

There are four functions declared and called in the script of the following HTML page:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>

           function func1() {
                var x;
                console.log(`x = ${x}`);
           }

           function func2() {
                var y;
                y += 31;
                console.log(`y = ${y}`);
           }

           function func3() {
                var z = null;
                console.log(`z = ${z}`)
           }

           function func4() {
                'use strict';
                t = 4;
                console.log(`t = ${t}`);
           }

           func1();
           func2();
           func3();
           func4();
      </script>
</body>
</html>

Which function will cause an error?

func1
func2
func3
func4

Explanation

func4 will cause an error.

The script starts by declaring four functions and then calling them. The variables are declared using the var keyword (except t). Some of the variables are initialized, and others are not.

One key element in func4() is the strict mode. This mode can be applied to files and to functions. Strict mode changes the JavaScript semantics and makes silent errors come to life. In general, JavaScript is forgiving when it comes to using variables before declaring them. In func4(), t was initialized before it was declared. But since the code in func4() is running in strict mode, this is no longer allowed. The code will cause the following error:

ReferenceError: assignment to undeclared variable t

Strict mode also has an impact on performance. That is why it is important to experiment code with and without strict mode.

The rest of the functions have uninitialized variables, but they will not cause errors. Here is what will be displayed in the console from each function:

func1()
x = undefined

func2()
y = NaN

func3()
z = null

Note that adding a number to an uninitialized variable will result in NaN (not a number).

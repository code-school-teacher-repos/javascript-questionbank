Interact with the Document Object Model
SubObjective:

Identify and handle HTML events

You have the following HTML on a page:

<div id="result" class="format-output"
      onmouseover="mouseOverAction(event)"
      onmouseout="mouseOutAction(event)"
      onmousemove="mouseMoveAction(event)"></div>
<div id="output1" style="color:red"></div>
<div id="output2" style="color:green"></div>

You need to record the user's mouse coordinates as it moves over the "result" div.

Which event function should you use?

function mouseOverAction(event) {       document.getElementById("output1").innerHTML = `(${event.X},${event.Y})`; }
function mouseOutAction(event) {       document.getElementById("output1").innerHTML = '(0,0)';       document.getElementById("output2").innerHTML = '(0,0)'; }
function mouseOverAction(event) {       document.getElementById("output1").innerHTML = `(${event.clientX},${event.clientY})`; }
function mouseMoveAction(event) {       document.getElementById("output2").innerHTML = `(${event.clientX},${event.clientY})`; }

Explanation

The function that records the mouse coordinates as it moves over the "result" div is:

function mouseMoveAction(event) {
      document.getElementById("output2").innerHTML = `(${event.clientX},${event.clientY})`;
}

When the mouse is outside the "result" div, output1 and output2 show (0,0). That will also be the case on the mouseout event. The mouse is outside the div and no coordinate tracking is taking place.

However, as the user moves the mouse inside the div, the mousemove event fires and the clientX and clientY values are recorded on every move.

You should not use a function that uses the mouseover event. The mouseover event is fired only once when the mouse is over the div. It never records any additional coordinates as the user is moving the mouse inside the div.

The appropriate values are clientX and clientY. The answer option that uses X and Y would result in 'undefined' being displayed.

An HTML page contains the following:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           do {
                var input = prompt("Enter a value");
                var acceptable = confirm("Continue?");
                var theType = determineType(input);
                console.log("The type is: " + theType);
           } while (acceptable)

           function determineType(input) {
                if (!input)
                     return 'undefined';
                switch (input.constructor) {
                     case String: return "string with value ' " + input + " ' ";
                     case Number: return "number with value " + input;
                     case Date: return "date with value " + input;
                     case RegExp: return "regular expression with value " + input;
                }
           }
      </script>
</body>
</html>

You load the page in the browser, and you enter these values before canceling out of the confirm dialog: test, 5, 8/11/2017, and /^abc$/.

What will be displayed in the console window?

The type is: undefined The type is: undefined The type is: undefined The type is: undefined
The type is: string with value 'test' The type is: number with value 5 The type is: date with value 8/11/2017 The type is: string with value '/^abc$/'
The type is: string with value 'test' The type is: number with value 5 The type is: date with value 8/11/2017 The type is: regex with value /^abc$/
The type is: string with value 'test' The type is: string with value '5' The type is: string with value '8/11/2017' The type is: string with value '/^abc$/'

Explanation

The console window displays the following:

The type is: string with value 'test'
The type is: string with value '5'
The type is: string with value '8/11/2017'
The type is: string with value '/^abc$/'

The script starts with a do-while loop that keeps iterating as long as "acceptable" is true. The loop prompts the user for a value. The result is stored in input. Then the confirm dialog asks the user whether to continue. Based on the user's selection of "ok" or "cancel", the value stored in "acceptable" will be true or false respectively.

The entered value is passed to a function named determineType() that checks the type of the input using the constructor. In object-oriented terminology, a constructor is the initializer or the function that constructs that specific type. The objects that you are comparing against here are: String, Number, Date and RegExp.

The switch statement (which is like an if statement) looks at the value of the constructor and finds a matching case. Every type has an associated statement being returned.

Based on the requirement, the user enters test, 5, 8/11/2017, and /^abc$/ before canceling the dialog. These values are a string, a number, a date, and a regular expression. However, it is important to note that the value returned by the prompt is always a string regardless of the structure of what has been entered. Therefore, if the user enters '1/1/2018', this is considered a string, not a date. With that in mind, the call to the determineType() function always matches the String case.

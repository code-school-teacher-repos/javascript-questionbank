Interact with the Document Object Model
SubObjective:

Complete and debug code that outputs to an HTML document

Which set of code would display the following on the screen?

Programming with JavaSQL Step by Step
JavaScript, the Easy Way
Python Revealed!

document.writeln("Programming with Java"); document.writeln("SQL Step by Step"); document.writeln("JavaScript, the Easy Way"); document.writeln("Python Revealed!");
document.write("Programming with Java"); document.write("SQL Step by Step"); document.write("JavaScript, the Easy Way"); document.write("Python Revealed!");
document.writeln("Programming with Java"); document.write("SQL Step by Step"); document.write("JavaScript, the Easy Way"); document.writeln("Python Revealed!");
document.write("Programming with Java"); document.writeln("SQL Step by Step"); document.writeln("JavaScript, the Easy Way"); document.write("Python Revealed!");

Explanation

The code that displays the correct output on the screen is:

document.write("Programming with Java");
document.writeln("SQL Step by Step");
document.writeln("JavaScript, the Easy Way");
document.write("Python Revealed!");

When writing content with the document object, there is a difference between using write() and writeln(). While write() writes text to the document stream that was opened with document.open(), the writeln() function does the same thing, but adds a newline character to the document.

In the output shown in the question, the following lines were attached:

Programming with JavaSQL Step by Step

That means the line that writes "Programming with Java" uses the write() function. But then "JavaScript, the Easy Way" was on a separate line, which indicates that the function that writes "SQL Step by Step" used the writeln() function. Since "Python Revealed!" was on a separate line as well, writeln() is also used to write the "JavaScript, the Easy Way" text.

In the other options, the write() and writeln() functions are used in ways that the output will not match the one given in the requirement.

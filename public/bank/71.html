SubObjective:

Complete and debug loops

Which code block will display the alert exactly once?

var num; var items = [35, 29, 8, 10, 25, 12, 45]; for (; num < 10; num++) {       items.forEach(function (item) {            if (item === num) {                 alert('Got it!')            }       }); }
while (true) {       var result = parseInt(prompt("Enter a value"));       if (result % 2 == 0) {            alert('Got it!');       } }
var x = 25; while (x > 10) {       if (x % 2 != 0)            alert('Got it!');       x--; }
do {       alert('Got it!'); } while (undefined);

Explanation

The code block that displays the alert only once is:

do {
      alert('Got it!');
} while (undefined);

The reason is that do-while loops test the condition after the body, which makes the loop execute at least once. In this case, the condition is undefined, which makes it fail right away, but the alert has been displayed before that condition is evaluated.

In the following case, num was declared outside the loop and never initialized. That gave the variable a value of undefined. With that, the loop condition is false, and the loop does not execute. There are no alerts displayed with this code block.

var num;
var items = [35, 29, 8, 10, 25, 12, 45];
for (; num < 10; num++) {
      items.forEach(function (item) {
           if (item === num) {
                alert('Got it!')
           }
      });
}

The following while loop has the condition always true, which makes it an infinite loop. There is no way to tell the number of times the alert will show. It all depends on the value entered by the user being even or odd.

while (true) {
      var result = parseInt(prompt("Enter a value"));
      if (result % 2 == 0) {
           alert('Got it!');
      }
}

In the following case, the alert will show many times. x is going down from 25 to 11. Every time the value is not even, the alert will show.

var x = 25;
while (x > 10) {
      if (x % 2 != 0)
           alert('Got it!');
      x--;
}

Interact with HTML Forms
SubObjective:

Describe the form submission process

You are creating the form shown in the work area. If the user enters "James&" in the first name and "My Last Name!" in the last name, the data submitted to the server will be encoded as:

firstName=James%26&lastName=My+Last+Name%21

You need to complete the form. What should you do? To answer, select the appropriate options from the drop-down menus.

Explanation

The correct method and enctype are:

method="post" enctype="application/x-www-form-urlencoded"

The enctype attribute is only used with the post method, so the method="get" option is not applicable. With the post method, there are three enctypes:

application/x-www-form-urlencoded: This is the default. With this option, all characters are encoded. All spaces are replaced with '+', and the special characters are converted to ASCII HEX.

multipart/form-data: This is used with a file upload control, and no characters are encoded.

text/plain: With this type, the special characters are not encoded, and the spaces are replaced with '+'.

Looking at the submitted data in the requirement:

firstName=James%26&lastName=My+Last+Name%21

It is clear that the spaces are replaced with '+' and there are HEX values for the special characters entered by the user after the first and last name:

First name: James&
Last name: My Last Name!

Therefore the enctype must be: application/x-www-form-urlencoded.

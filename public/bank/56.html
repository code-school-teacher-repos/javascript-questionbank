Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug a function that accepts parameters and returns a value

You are debugging the following code:

let name = "Tester, JavaScript";
let recordCount = 100;
let book = {
      name: "Fun with programming",
      author: "unknown",
      pageCount: 557
}

function takeAction(name, recordCound, book) {
      let first, last;
      let index = name.indexOf(',');
      if (index >= 0) {
           first = name.substring(index + 2);
           last = name.substring(0, index);
           recordCound = 50;
      }
      else
           recordCound = 75;

      if (book.author === 'unknown') {
           book.author = 'Bryan';
           recordCound = 125;
      }
      else
           recordCound = 0;
}

console.log(`record count: ${recordCount}`);
takeAction(name, recordCount, book);
console.log(`record count: ${recordCount}`);

What will be displayed in the console?

record count: 100 record count: 75
record count: 100 record count: 50
record count: 100 record count: 125
record count: 100 record count: 100

Explanation

The following will be displayed in the console:

record count: 100
record count: 100

The function takeAction() takes three parameters as follows:

function takeAction(name, recordCound, book) {...}

Passing parameters into functions can be by value and by reference. Basic types, such as numeric values, are passed by value, and complex types, such as objects, are passed by reference. In the example above, recordCount is passed by value and book is passed by reference. By value means only a copy of the variable is passed to the function, not the original variable. By reference, on the other hand, means that an actual reference of the variable in memory is passed to the function.

The takeAction() function inspects the name. If there is a coma, it strips out the first and last and places them in variables. Along the way it manipulates the record count. Along the same token, it checks the content of the book's author and changes the record count.

Knowing that recordCount is passed by value, all the changes are actually made to the copy of recordCount, not the variable itself. It is true that the copy of recordCount changed to 50 and then 125 inside the function, but by the time the function exits, recordCount is actually 100. Therefore, recordCount is 100 before the function call and 100 after the function call.

The following will not be displayed in the console:

record count: 100
record count: 50

After stripping out the first and last name, recordCount's copy was changed to 50. Then it changed to 125 in the second if block. However, as illustrated above, only the copy changed and recordCount is back to 100 after the function terminates.

The following output will not be displayed in the console:

record count: 100
record count: 125

Before the function terminates, recordCount's copy is 125 because the author's name is 'unknown'. After the function finishes, recordCount retains its original value (100). So, both of the console statements in the following code snippet print 100:

console.log(`record count: ${recordCount}`);
takeAction(name, recordCount, book);
console.log(`record count: ${recordCount}`);

The following statements will not be displayed in the console:

record count: 100
record count: 75

The copy of recordCount never changed to 75 inside the function because the if (index >= 0) is true and the if block executes, not the else block. Regardless of what happens inside the function, by the time the function finishes, recordCount changes back to 100.

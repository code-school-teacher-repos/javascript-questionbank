Interact with the Document Object Model
SubObjective:

Complete and debug code that outputs to an HTML document

You have the following code:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
      <style>
           .format-output {
                width: 300px;
                height: 100px;
                border: 1px solid blue;
                font: bold 20px "courier new";
                text-align: center;
           }
      </style>
</head>
<body>
      <div id="result1" class="format-output"></div>
      <div id="result2" class="format-output"></div>
      <script>
           document.getElementById("result1").innerHTML = "<b>Hello World</b>";
           document.getElementById("result2").innerText = "<b>Hello World</b>";
      </script>
</body>
</html>

What will be displayed in the div with id result1 and the div with id result2?

The first div displays "Hello World" in bold, and the second div displays "<b>Hello World</b>".
They both display "<b>Hello World</b>".
They both display "Hello World" in bold.
The first div displays ""<b>Hello World</b>", and the second div displays "Hello World" in bold.

Explanation

The first div displays "Hello World" in bold, and the second div displays "<b>Hello World</b>".

In the following statement, the HTML content of the element is changed, and since the content already contains HTML, it will get interpreted by the browser and displayed in bold:

document.getElementById("result1").innerHTML = "<b>Hello World</b>";

In the second statement, the content of the div is changed as well, but in this case using inner text:

document.getElementById("result2").innerText = "<b>Hello World</b>";

innerText encodes and decodes the text automatically. It retrieves and sets the content inside the tag as plain text. In other words the <b> is not interpreted and bold, and the text will not be rendered as bold.

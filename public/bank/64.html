Implement and Analyze Decisions and Loops
SubObjective:

Evaluate expressions that use logical and comparison operators

You have the following script block:

<script>
      var x = 42;
      var y = !x;
      var exp = (Number(undefined) == NaN) && (Boolean(null) == false) && (!y == true) && (x++ === 42);
      console.log(exp);
</script>

Which expression causes exp to evaluate to false?

Boolean(null) == false
!y == true
Number(undefined) == NaN
x++ === 42

Explanation

The expression that is causing exp to evaluate to false is:

Number(undefined) == NaN

exp is a combination of expressions joined with &&. That means all the expressions need to evaluate to true in order for exp to be true.

Starting with the first expression, converting undefined to a number will result in NaN. However, a NaN is never equal to NaN. Therefore this expression is false. If you need to test whether a result is NaN, you should use the function isNaN(). So, isNaN(Number(undefined)) is true.

Converting a null to a Boolean will be false. The expression Boolean(null) == false is true.

Since x is 42, !x is false and y is false. That means !y is true and !y == true is true.

x is 42. x++ === 42 is true because x gets evaluated before it is incremented (++ is after, not before x).

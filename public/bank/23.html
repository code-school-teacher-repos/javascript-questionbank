You have the following web page accessed at http://localhost/index.html

<!DOCTYPE html>
<html>
<head>

 <title>Intro to JavaScript</title>
 <meta charset="utf-8"/>
</head>
<body>

 <script>
 try {
 showMeTheLog();
 }
 catch (e) {
 logIt(e);
 }
 
 // FUNCTION TO LOG EXCEPTIONS
 function logIt(e) {
 console.log(`${e instanceof TypeError}::${e.message}::${e.name}`);
 }
 </script>
</body>
</html>

What will be displayed in the console when the page is loaded in the browser?

Nothing will be logged
false::showMeTheLog is not defined::ReferenceError
true::undefined is not an object::TypeError
true::undefined has no properties

Explanation

The following will be displayed in the console:

true::undefined has no properties::TypeError::http://localhost:33931/SubObjective_1_4_5.html
Finally

The function showMeTheLog() tries to call the parseInt() on undefined. parseInt() is a JavaScript function that converts a string into an integer. undefined does not have such a function. That will throw an error. The catch blocks in this case are catching errors based on types. For example, if the error is a range error, the associated catch block is executed. There are other types of errors such as: EvalError, InternalError, RangeError, ReferenceError, SyntaxError, and TypeError. The catches in the example are only handling TypeError, SyntaxError, and RangeError.

Since undefined does not have parseInt() as a function or property, a TypeError will be a match. In other words, the error that took place is an instance of TypeError. Therefore, the following will show in the console:

true::undefined has no properties::TypeError::http://localhost:33931/SubObjective_1_4_5.html
Finally

A RangeError is thrown when accessing an element out of range. Here is an example:

var elements = [1,2,3,4,5];
var index = 15;
if (index < 0 || index > 4) {
throw new RangeError('range is out of bounds');

With a RangeError, the following can be displayed in the console:

false::range is out of bounds::RangeError::http://localhost:33931/SubObjective_1_4_5.html
Finally

The "false" at the beginning of the statement is caused by the logIt() function because it starts with "e instanceof TypeError" and e in this case is a RangeError.

To display a SyntaxError, the following can be attempted:

eval('x = 1 y = 2');

Since the semicolon is missing before "y", the following will show in the console:

false::missing ; before statement::SyntaxError::http://localhost:33931/SubObjective_1_4_5.html
Finally

The logIt() function uses the back-tick (``) to show the content of the error. To evaluate a variable in the string, the following syntax is used: ${...}.

Interact with the Document Object Model
SubObjective:

Complete and debug code that locates, modifies, and adds HTML elements and attributes

You have the following page:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <div id="main">
           <p id="p1">This is a paragraph</p>
           <p id="p2">So is this one</p>
      </div>
      <div id="footer">
           <p>This is the footer</p>
      </div>
      <script>
           let element = document.getElementById('main').nextSibling;
           while (element) {
                console.log(element.nodeName);
                element = element.nextSibling;
           }
      </script>
</body>
</html>

What will be displayed in the console when the page is loaded in the browser?

DIV P SCRIPT
DIV SCRIPT
#text DIV #text SCRIPT
DIV

Explanation

When the page loads in the browser, the following will be displayed in the console:

#text
DIV
#text
SCRIPT

nextSibling is a read-only property that gets the node immediately after the one specified. In this scenario, the specified one if the div with the main id. The one immediately after it is the div with the id footer.

However, what is not immediately clear is the whitespace that exists in the markup between tags. That whitespace is represented with #text in the console output. So the output states that there is whitespace, then the DIV with the footer id, then some more whitespace, and then the JavaScript script tag.

Note that if you had used nextElementSibling, it would skip the whitespace nodes.

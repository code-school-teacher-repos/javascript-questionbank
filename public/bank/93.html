Interact with the Document Object Model
SubObjective:

Complete and debug code that locates, modifies, and adds HTML elements and attributes

The body of an HTML page contains the following:

<div>Lists</div>
<div>
      <ol>Books
           <li>Book 1</li>
           <li>Book 2</li>
           <li>Book 3</li>
      </ol>
</div>
<div>
      <ol>Colors
           <li>Red</li>
           <li>Green</li>
           <li>Blue</li>
           <li>Orange</li>
      </ol>
</div>
<div id="result"></div>
<script>
      var items = document.getElementsByTagName("ol");
      for (var i = 0; i < items.length; i++) {
           items[i].firstChild.textContent = "Items";
      }
</script>

What is displayed when the page is loaded in the browser?

Lists Books 1.Items 2.Book 2 3.Book 3 Colors 1.Items 2.Green 3.Blue 4.Orange
Lists Books 1.Book 1 2.Book 2 3.Book 3 Colors 1.Items 2.Green 3.Blue 4.Orange
Lists Books 1.Items 2.Book 2 3.Book 3 Colors 1.Red 2.Green 3.Blue 4.Orange
Lists Items 1.Book 1 2.Book 2 3.Book 3 Items 1.Red 2.Green 3.Blue 4.Orange

Explanation

The page will display as follows:

Lists

Items
1.Book 1
2.Book 2
3.Book 3

Items
1.Red
2.Green
3.Blue
4.Orange

The page displays two ordered lists, one for books and one for color. The script gets the element ol by tag name. That returns two of them: the books and the colors. The for loop iterates over the two sets and for each one of them gets the first child. The first child is the actual content of the tag (Books and Colors), not the first list item (Book 1 and Red). That means Books and Colors get overwritten with the content Items, but the rest of the lists will not be impacted.

The options that show the list items being replaced with Items are not correct because the firstchild call returns the text of the ordered list, not the first list item.

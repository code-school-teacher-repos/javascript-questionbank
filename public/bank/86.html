You are developing an application that supports the IE11 and IE Edge browsers.

As the user is typing characters in the input box, you need to echo the characters in the "result" div.

Which code set should you use?

<input type="text" onkeydown="keydownAction(event)"> function keydownAction(event) {       var value = event.keyCode;       document.getElementById("result").innerHTML += String.fromCharCode(value); }
<input type="text" onkeydown="keydownAction(event)"> function keydownAction(event) {       var value = event.which;       document.getElementById("result").innerHTML += String.fromCharCode(value); }
<input type="text" onkeydown="keydownAction()"> function keydownAction(event) {       document.getElementById("result").innerHTML += event.keyCode; }
<input type="text" onkeydown="keydownAction(event)"> function keydownAction(event) {       document.getElementById("result").innerHTML += event.which; }

Explanation

You should use the following code set:

<input type="text" onkeydown="keydownAction(event)">
function keydownAction(event) {
      var value = event.which;
      document.getElementById("result").innerHTML += String.fromCharCode(value);
}

The keydownAction()function handles the onkeydown event.

event.which returns the Unicode key code for the corresponding character. For example, if the user enters the letter 'a', the value will be 65. To return the actual letter, the function String.fromCharCode() needs to be applied on the value.

keyCode also returns the key, not the letter. But this was the case for IE8 and earlier versions. Because the requirement states IE11 and IE Edge, you should use which instead.

The code sets that return only keyCode or which are not correct because they do not convert the key to a letter using the String.fromCharCode() function.

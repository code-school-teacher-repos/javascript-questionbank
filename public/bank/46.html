Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug code that uses objects

What is the result of loading the following page in the browser?

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           var book = new Object();
           book.title = 'JavaScript Programming';
           book.author = 'Lisa Doe'
           book.pageCount = 495;
           printIt(book);

           function printIt(b) {
                console.log(b['title']);
                console.log(b['author']);
                console.log(b['pageCount']);
                console.log(b['publisher']);
           }
      </script>
</body>
</html>

An error will display because book is an object, not an array.
The following will be displayed in the console: JavaScript Programming Lisa Doe 495 null
An error will display because publisher is not a property of book.
The following will be displayed in the console: JavaScript Programming Lisa Doe 495 undefined

Explanation

The following will be displayed in the console:

JavaScript Programming
Lisa Doe
495
undefined

book is declared as an object. It has the following properties: title, author, and pageCount.

The function printIt() logs the content of the book object in the console. Object properties are accessed using the dot notation, but they can also be accessed using the bracket notation. Objects are called associative arrays. You can use the string value of the property to index the object and retrieve the content. Therefore indexing the object in that way will not cause an error.

In addition, unassigned properties of an object (such as publisher) are undefined, not null, and they will not cause an error either.

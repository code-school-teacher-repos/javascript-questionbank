You load the following page in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
      <style>
           .format-output {
                width: 300px;
                height: 100px;
                border: 1px solid blue;
                font: bold 20px "courier new";
                text-align: center;
           }
      </style>
</head>
<body>
      <div id="result" class="format-output"></div>
      <script>
           var student = { 'name': 'John', 'course': 'CS-101' };
           var students = {
                first: { 'name': 'John', 'course': 'CS-101' }
           };
           var courseName = new String('JavaScript');
           var studentCount = new Number(34);
           var openEnrollment = true;

      var outcome = ' ';
           if (student == students.first)
                outcome = 'Student ';
           if (courseName === 'JavaScript')
                outcome += 'JavaScript ';
           if (studentCount == 34)
                outcome += 'Count ';
           if (undefined || null || !openEnrollment || student)
                outcome += 'Something ';

           document.getElementById("result").innerHTML = outcome;
      </script>
</body>
</html>

Which text will be displayed on the screen?

Student JavaScript Count
Student JavaScript Count Something
Count Something
JavaScript Count

Explanation

Count Something will be displayed on the screen.

The script starts with the following declarations:

var student = { 'name': 'John', 'course': 'CS-101' };
var students = {
      first: { 'name': 'John', 'course': 'CS-101' }
};
var courseName = new String('JavaScript');
var studentCount = new Number(34);
var openEnrollment = true;

The if statements test conditions and accumulate string in the variable 'outcome'. Although the student object and the students.first object contain the same information, they are not equal. Objects in JavaScript are references, and the references are not equal. They are in different places in memory.

The second condition tests whether the courseName variable is 'JavaScript'. Even if the strings are with the same content, you are using the === operator, which tests for content and type. The string literal 'JavaScript' is not equal to the string object String('JavaScript').

The studentCount object is compared to 34 using the == operator. In this case, a type casting is taking place and studentCount is converted to 34, which is equal to 34, and 'Count ' is printed on the screen.

The following condition uses or (||), which indicates that one true item within the expression is enough to make the whole expression true. undefined, null and !openEnrollment are all false. But student is true. That makes the expression true and 'Something ' is also printed on the screen:

undefined || null || !openEnrollment || student

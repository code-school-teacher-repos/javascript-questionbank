Interact with the Document Object Model
SubObjective:

Complete and debug code that outputs to an HTML document

An HTML page contains the following:

<div id="result"></div>
<script>
//Add code here
</script>

You need to add code to the <script> tag so that the current date is displayed on the screen and refreshed every second.

Which code snippet should you use?

setInterval(function() {      let d = new Date();      document.getElementByClassName("result").innerHTML = d; }, 1000);
setInterval(function() {      let d = new Date();      document.getElementById("result").innerHTML = d; }, 10000);
setInterval(function() {      let d = new Date();      document.getElementById("result").innerHTML = d; }, 1000);
function printDate() {      for (var i = 1; i <= (24*60*60); i++) {           let d = new Date();           document.getElementById("result").innerHTML = d;       } } printDate();

Explanation

The code snippet that must be added to the <script> tag is:

setInterval(function() {
      let d = new Date();
      document.getElementById("result").innerHTML = d;
}, 1000);

The JavaScript setInterval() function repeatedly calls a code snippet with a fixed time interval. The time interval is given in milliseconds. The first parameter to setInterval() is the function:

function() {
      let d = new Date();
      document.getElementById("result").innerHTML = d;
}

The function retrieves the current date and places it in the <div> tag identified by its ID result.

The second parameter to setInterval() is 1000, which is the time delay in milliseconds. This value converts to 1 second. With that code snippet, the current date (down to the second) is displayed on the screen and refreshed every second.

You should not use the following code snippet because although it retrieves the current date, it refreshes every 10 seconds (10000):

setInterval(function() {
      let d = new Date();
      document.getElementById("result").innerHTML = d;
}, 10000);

You should not use the following code snippet because it retrieves the result by class name (getElementByClassName("result")) instead of by ID. For that reason, nothing will be displayed. The <div> tag does not have a CSS class named result, but it does have an ID named result:

setInterval(function() {
      let d = new Date();
      document.getElementByClassName("result").innerHTML = d;
}, 1000);

You should not use the following code snippet because even if the for loop is iterating 86400 times, only one display shows on the screen, and that is not refreshed every second:

function printDate() {
      for (var i = 1; i <= (24*60*60); i++) {
           let d = new Date();
           document.getElementById("result").innerHTML = d;
      }
}
printDate();

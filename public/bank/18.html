SubObjective:

Evaluate the use of inline and external scripts

You have the following HTML page and its referenced Common6.js file:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
      <script src="Scripts/Common6.js"></script>
</head>
<body>
      <div id="result"></div>
      <script>
           function doAction(data, size) {
                if (!data || size <= 0)
                     return -1;
                var newData = data.split(' ');
                var result;
                for (var i = 0; i < size; i++) {
                     result += newData[i];
                }
                return result;
           }
      </script>

      <script>
           var info = "1 2 3 4 5 6 7"
           document.getElementById("result").innerHTML = doAction(info, 5);
      </script>
</body>
</html>

Common.js



function doAction(data) {
      if (!data)
           return -1;
      var newData = data.split(' ');
      var result = 0;
      for (var i = 0; i < 5; i++) {
           result += newData[i];
      }
      return result;
}

What will be displayed in the browser when previewing the page?

012345
15
12345
undefined12345

Explanation

The browser will display the following: undefined12345. The HTML page references Common6.js file in the header. This file already contains a function named doAction(). This function takes a collection of elements as a parameter (data). It splits the collection using a space and stores the resulting collection in another variable (newData). Then, it loops 5 times, reading each element from newData and accumulating them in a result.

On the other hand, the body of the HTML page contains another function named doAction() that takes two parameters, the data and the size. It splits the data using a space, storing the outcome in newData, then looping over the collection "size" times and accumulating the result in "result". The second script on the page creates a string containing "1 2 3 4 5 6 7" and passing that to doAction(). The question is: which doAction() will be called? The one with two parameters, or the one with one parameter?

It turns out that functions in JavaScript are references. So when the first doAction() was encountered when loading the Common6.js file, the reference was pointing to that function. But then when the script in the body was loaded, another doAction() was encountered. Since it is the same name, now doAction() references the new set of code in the body of the HTML page. Therefore, the number of parameters in the function does not make a difference (like it does in other languages such as Java or C#). Since the number collection is passed as a string to the function, looping over the collection and accumulating the elements is a string accumulation or concatenation, NOT a number addition. Therefore the result is not 15. Looping over and accumulating "1 2 3 4 5 6 7" results in "1234567". However, since the function doAction() was called with 5 as a second parameter, the result is "12345". Furthermore, notice that the variable result was declared as var result; which is not initialized. That makes the initial value "undefined". That makes the final result in the browser window "undefined12345".

SubObjective:

Complete or debug code that uses assignment and arithmetic operators

While investigating a bug in an existing application, you come across the code shown below:

<div id="result" class="format-output"></div>
<script>
      var studentCount = 14;
      var newCount = studentCount++ + ++studentCount - (17 % 2)
      document.getElementById("result").innerHTML = newCount;
</script>

What will be displayed on the screen?

29
28
22
30

Explanation

The script will display 29. ++ and % are JavaScript operators. While ++ is used to increment a variable, % is the remainder operator. In this scenario, studentCount is initialized to 14. In the following statement, the initial value of 14 is read. But then studentCount++ evaluates to 15, and ++studentCount increments the value to 16.

var newCount = studentCount++ + ++studentCount - (17 % 2);

Breaking the right-hand side into three pieces, the values are:

studentCount++ = 14
++studentCount = 16
(17 % 2) = 1

Therefore, the value of newCount is 14+16-1 = 29.

The value of the expression is not 30 because studentCount++ + ++studentCount does not evaluate to 31. The location of the ++ operator is crucial in reading the value of a variable. If the ++ is after the variable, the content of the variable is read first, and then the increment takes place. If the ++ is before the variable, the increment takes place first, and then the content is read.

For the same reasons above, the expression does not evaluate to 29.

The % operator is the remainder of the division. Therefore 17%2 evaluates to 1, not 8. Therefore, the value of the whole expression does not evaluate to 22.

Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug code that uses objects

You have the following script:

<script>
var date1 = new Date('January 19, 2018 1:31:05');
var date2 = new Date(2018, 0, 15, 1, 30, 29);
var value = date1.getTime() - date2.getTime();

console.log(`There are ${value2} days to be ready for the game`)
</script>

Which line would correctly complete the code?

var value2 = Math.floor(value / (1000*60*60*24));
var value2 = Math.floor(value%24 / (date1.getMilliseconds()));
var value2 = (value*60*60) / 1000;
var value2 = Math.floor(value / date1.getSeconds());

Explanation

The missing line is:

var value2 = Math.floor(value / (1000*60*60*24));

value represents the difference between the two dates using the getTime() function, which is the number of milliseconds since January 1, 1970. In order to get the number of days, you need to divide the value by: (1000*60*60*24).

The values are explained below:

1000 is to convert the milliseconds to seconds.
60 is for the number of seconds in a minute.
60 is for the number of minutes in an hour.
24 is for the number of hours in the day.

You use the Math.floor() function to find the integer representation and remove any decimals.

You should not use the following line:

var value2 = Math.floor(value / date1.getSeconds());

date1.getSeconds()returns 5, and the statement would be "There are 69127200 days to be ready for the game".

You should not use the following line:

var value2 = (value*60*60) / 1000;

value*60*60 returns 1244289600000, and the statement "There are 1244289600 days to be ready for the game" does not reflect the correct conversion to days.

You should not use the following line:

var value2 = Math.floor(value%24 / (date1.getMilliseconds()));

Since the number of milliseconds is not specified in the constructor Date('January 19, 2018 1:31:05'), the value is 0, and the statement will be "There are NaN days to be ready for the game" considering that dividing by 0 returns NaN.

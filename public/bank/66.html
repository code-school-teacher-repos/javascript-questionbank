Complete and debug decision statements

Which two code blocks would cause an alert to show on the screen?

if (undefined)       alert('undefined satisfied!');
if (Infinity)       alert('Infinity satisfied!');
if (NaN)       alert('NaN satisfied!');
if (+0)       alert('+0 satisfied!');
if (isNaN("32"))       alert('isNaN satisfied!');
if (new String("test"))       alert('new String satisfied!');
if (null)       alert('null satisfied!');

Explanation

The code blocks that would cause an alert to show in the screen are:

if (new String("test"))
      alert('new String satisfied!');

if (Infinity)
      alert('Infinity satisfied!');

For the body of an if statement to execute, the condition between the parentheses must evaluate to true.

A string object containing text (in this case "test") evaluates to true, and the alert will display. On the other hand, if the string was empty, that would evaluate to false.

Infinity is a large number, and since all numbers (except 0) are considered true in a condition, the alert associated with that condition will also display.

There are many reserved words that evaluate to false. This includes: undefined, null and NaN. All zeros (-0, 0, and +0) are considered false while evaluated in a Boolean condition.

Although "32" is included within quotes, it is converted to a number, and the expression isNaN("32") is false. The associated alert will not display for this condition.

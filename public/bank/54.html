You preview the following page in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
      <style>
           .format-output {
                width: 300px;
                height: 100px;
                border: 1px solid blue;
                font:bold 20px "courier new";
                text-align:center;
           }
      </style>
</head>
<body>
      <div id="result" class="format-output"></div>
      <script>
           let names = ['Apple', 'Orange', 'Banana', 'Kiwi'];

           {
                let names = ['Blue', 'Green', 'Orange', 'Yellow', 'Red'];

                     function findIndex(element) {
                          if (!element) return;
                          for (var i = 0; i < names.length; i++) {
                               if (names[i].toLowerCase() === element.toLowerCase())
                                    return i;
                               }
                          }
                     }

           function getBefore(i) {
                if (i <= 0) return;
                return names[i - 1];
           }

           names.sort();
           var index = findIndex('orange');

           document.getElementById("result").innerHTML = getBefore(index);
      </script>
</body>
</html>

Which word will be displayed on the screen?

Yellow
Green
Banana
Kiwi

Explanation

The word Banana will be displayed on the screen.

There are two key elements to consider in this code: (1) The array names is declared twice and (2) The function findIndex() is declared in a separate block. The function findIndex(), as the name implies, finds the index of the element in the array ignoring case. The getBefore() function returns the element right before the one with the specified index.

Both arrays are declared with the let keyword, which is scope aware. In other words, the outer declaration will not interfere with the inner one. With that in mind, the sort() method is being applied to the outer array.

After sorting the outer names array, findIndex() is called. It is important to note that the loop in this function is looping over the color names, not the fruit names because it is using the array in the current scope. The value returned is 2 since "orange" is at that index in the color array.

When the function getBefore() is being called with the value 2, it is currently looking at the fruit array because, again, this is the array in the current scope (the script scope, not the block scope of findIndex()) . Since that array was sorted, it has the values "Apple", "Banana", "Kiwi", and "Orange". "Kiwi" is at index 2. The one before it is "Banana", which is being returned.

You are debugging a web application. You find the JavaScript function that is likely the source of the bug.

You need to stop the code execution at the beginning of the function and step through the code line by line.

What should you do?

Add a debugger statement at the beginning of the function.
Add console.log statements to the function.
Add a debugger statement at the beginning of the function and open the developer tools.
Add console.table statements to the function.

Explanation

You should add a debugger statement at the beginning of the function and open the developer tools. While debugging JavaScript code, a developer can add debugger statements in the code to stop the execution at that point or add log statements that display the content of the variables in the console. In this scenario, the user needs to step through the code line by line. To do so, a debugger statement is needed. However, adding the debugger statement is not sufficient. If there is no debugging functionality, the statement has no effect. Therefore, adding the debugger statement and opening the developer tools (F12) will help the developer stop at that statement and step through the code.

You should not add console.log statements to the function. Adding log statements to the function is helpful in displaying the content of the variables to the console. However, that will not be enough to stop debugging at the function in question and step through the code line by line.

You should not add console.log statements to the function. console.table is similar to console.log, but it displays its content in a table format, and it will not be enough for this case.

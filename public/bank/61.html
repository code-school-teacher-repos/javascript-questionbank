Implement and Analyze Decisions and Loops
SubObjective:

Evaluate expressions that use logical and comparison operators

You have the following declarations:

var obj1 = {'name':'tester'}, obj2 = {'name':'tester'};
var exp1 = obj1 == obj2;
var exp2 = obj1 === obj2;
var exp3 = NaN === NaN;
var exp4 = 'test' == new String('test');

You need to complete the code so "Hello, world!" will show in the browser when the page loads.

Which expression should you use? To answer, select the appropriate expression from the drop-down menu.

Explanation

exp4 will evaluate to true, and that causes "Hello, world!" to be displayed in the browser window.

Strings are equal when they have the same sequence of characters and they are the same length. In this scenario, String('test') will be 'test', which means 'test' == 'test', and that evaluates to true. Having an expression evaluates to true in an "if" statement causes the body of the "if" statement to execute and make "Hello, world!" show in the browser. Since this is the only expression that evaluates to true, placing it in the blank will cause "Hello, world!" to display.

You should not select exp1 or exp2. The following declares two objects: obj1 and obj2:

var obj1 = {'name':'tester'}, obj2 = {'name':'tester'};

These objects contain the same member (name) with the same content (tester). However, JavaScript treats objects as references. In other words, obj1 and obj2 do not hold the actual contents. Rather, they hold a memory address to where that content is. That address is different for each one. For that reason, obj1 and obj2 are not equal, even when they are tested with the === operator. Therefore, exp1 being obj1 == obj2 is false and exp2 being obj1 === obj2 is also false. A false expression in the "if" block will cause the body of the "if" statement not to execute and "Hello, world!" will not be displayed.

NaN stands for Not a Number and it is not even equal to another NaN. So NaN == NaN is always false. With a false result in exp3, the body of "if" will not execute and "Hello, world!" will not show in the browser.

SubObjective:

Complete and debug code that uses built-in Math functions

The following page is loaded in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           var x = parseFloat(prompt("Please enter a value"));
           console.log(`The result is ${Math.round(Math.acos(x) * (180 / Math.PI))}`);
           console.log(`The result is ${Math.round(Math.cos(x))}`);
           console.log(`The result is ${Math.round(Math.asin(x) * (180 / Math.PI))}`);
           console.log(`The result is ${Math.round(Math.sin(x))}`);
           console.log(`The result is ${Math.round(Math.atan(x) * (180 / Math.PI))}`);
      </script>
</body>
</html>

The following is shown in the console:

The result is 0
The result is 1
The result is 90
The result is 1
The result is 45

Which value was entered by the user?

0.5
1
30
0

Explanation

In order to get the following result in the console, the use would enter 1 as the value:

The result is 0
The result is 1
The result is 90
The result is 1
The result is 45

The methods used are:

Math.round(): Rounds a number to the nearest integer.
Math.sin(): Returns the sine of an angle given in radians. Result between -1 and 1.
Math.cos(): Returns the cosine of an angle given in radians. Result between -1 and 1.
Math.asin(): Returns the arcsine of a number, which is between -1 and 1.
Math.acos(): Returns the arccosine of a number, which is between -1 and 1.
Math.atan(): returns the arctan of a number, which is between -PI/2 and PI/2 radians.

A value can be converted to degrees by multiplying the result with (180 / Math.PI). For example, the result of 90 can be explained as follows: You are finding the angle that has a sine of 1. Then the result is converted to degrees using (180 / Math.PI).That will be 90 degrees.

With the given function, if the user enters 0, the result would be:

The result is 90
The result is 1
The result is 0
The result is 0
The result is 0

For 30, the result would be:

The result is NaN
The result is 0
The result is NaN
The result is -1
The result is 88

Considering that the entered value (30) is not between -1 and 1, notice how the asin() and acos() calls return NaN (not a number).

For 0.5, the result would be:

Program with JavaScript Operators, Methods, and Keywords
SubObjective:

Complete and debug code that interacts with the Browser Object Model (BOM)

You are using the pushState() function in the history object as follows:

history.pushState(stateObject, "My Page", "test.html");

The stateObject is 1 MB in size.

You need to retain information needed by the user as long as the window is open.

What should you do?

Throw an InternalError.
Use the pushState() function as is.
Use localStorage instead.
Use sessionStorage instead.

Explanation

You should use sessionStorage instead. The sessionStorage function stores the data for only that session. In other words, the data is there until the user closes the tab or the window.

You should not use the pushState() function as is. The pushstate() function has three parameters: the state object, the title, and the URL. It is used to change the history entries. You store data in the state object and it gets serialized. But this data is stored in the user's disk and has a limit of 640k. Since the data you are storing is larger than the 640k (1 MB), you will not be able to serialize it. If you try, the method will throw an exception.

You should not use localStorage. When using localStorage, the data will not be deleted when the browser is closed.

You should not throw an InternalError because there are other options that allow you to store the data. You would throw an error because you do not have any other solution and you need to report the problem to the user. In this scenario, you have at least two solutions. The localStorage and sessionStorage functions are solutions to this same problem (very limited storage size).

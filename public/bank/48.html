Program with Variables, Data Types, and Functions
SubObjective:

Complete and debug code that uses objects

The following page is shown in the browser:

<!DOCTYPE html>
<html>
<head>
      <title>Intro to JavaScript</title>
           <meta charset="utf-8" />
</head>
<body>
      <script>
           var Shape = {
                type: 'Square',
                length: 10,
                width: 10,
                info: function() {
                     console.log(`${this.type}: ${this.length}x${this.width}`);
                }
           };

           var shape1 = Object.create(Shape);
           shape1.info();

           var shape2 = Object.create(Shape);
           shape2.width = 5;
           shape2.type = 'Rectangle';
           shape2.info();
      </script>
</body>
</html>

What will be displayed in the console?

Square: 10x10 Square: 10x10
Square: undefined Rectangle: undefined
Rectangle: 10x5 Rectangle: 10x5
Square: 10x10 Rectangle: 10x5

Explanation

The following will be displayed in the console:

Square: 10x10
Rectangle: 10x5

The Shape object is created with three properties (type, length, and width), and a function (info). The create()function is used to create objects using the Shape prototype.

shape1 is created, and its info is displayed in the console (Square: 10x10).
shape2 is also created using the Shape prototype. Changing the properties of shape2 (type and width) will not impact shape1. shape2.info() displays Rectangle: 10x5.

Since shape1's type is a Square and shape2's type is a Rectangle, the following output is not possible:

Rectangle: 10x5
Rectangle: 10x5

For the same reason, the following output is not possible either:

Square: 10x10
Square: 10x10

For the square, the width and the length are 10, and the rectangle's length has been adjusted to 5. When printing these values to the console, the output cannot be undefined.

Interact with HTML Forms
SubObjective:

Complete and debug code to perform input validation

You have the following form:

<form>
      <div>
           <label for="primaryEmail">Email:</label>
           <input type="email" id="primaryEmail" name="primaryEmail">
      </div>
      <div>
            <input type="submit" value="Submit">
      </div>
</form>

As the user is entering an email address, you need to provide instant feedback when the data is invalid by setting the input box background color to yellow and the text color to red.

Which style should you use?

input:focus:valid {      background-color: yellow;      color: red; }
input:focus:invalid {      background-color: yellow;      color: red; }
input#email:focus:invalid {      background-color: yellow;      color: red; }
input:valid {      background-color: yellow;      color: red; }

Explanation

You should use the following style:

input:focus:invalid {
     background-color: yellow;
     color: red;
}

This style targets an invalid input box, but only when it has focus. This is important especially on large forms because you do not need to turn the background color to yellow and text color to red for all input. That will confuse the user. This also matches the requirement because it provides instant feedback as the user is filling that particular field. This also means that setting the styles for input:invalid is incorrect.

You should not use input:valid {...} because it will give the user the wrong impression on valid data.

You should not use input#email:focus:invalid {...}. Although it is handling the focus case, it is looking for an input box with id 'email' (input#email). Such field does not exist on the form. The current id is 'primaryEmail'.
